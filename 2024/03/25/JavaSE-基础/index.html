Java跨平台

高级语言的执行步骤

  • 编程:Java程序员写的.java代码,c程序员写的.c代码,python程序员写的.py代码
  • 编译(即”翻译”):机器只认识01的机器语言,需要吧.java,.c,.py的代码转化成机器语言
  • 运行:让机器执行机器指令

高级语言的编译运行类型

  • 编译型:如c/c++语言,编写.c文件,然后编译为.obj文件,然后运行
  • 解释性:如python语言,.py文件按行翻译执行,不生成别的文件
  • 混合型(半编译+半解释):如java语言,先把.java文件整体编译成.class文件(javac),再按行把代码交给虚拟机运行(java),可以在不同平台的虚拟机中执行.class文件, 所以具有良好跨平台性

Java跨平台原理

=> 其是混合型高级语言, 先经过javac编译成为.class字节码文件, 再通过运行在各种操作系统上的jvm虚拟机运行

数据类型

Java中的数据类型分为基本数据类型 和 引用数据类型

本质区别是占用内存不同, 直接区别是数据取值范围不同

基本数据类型(4类8种)

  • 整数类型
    • byte -> 1Byte,[-128,127]
    • short -> 2Bytes,[-32768,32767]
    • int -> 4,[-2^31^,2^31^-1]
    • long -> 8,[-2^63^,2^63^-1],要使用long类型,就要加’L’后缀
  • 浮点数类型(小数类型)
    • float -> 4,要使用float类型,就要加’F’后缀
    • double -> 8
  • 字符类型
    • char -> 2
  • 布尔类型
    • boolean -> 1Byte,只有true或false

引用数据类型

基本使用

  • 若全是整数运算,除不尽,截取余数,保留整数位,如 10/3=3

类型转换

隐式转换(自动类型提升)

  • 作用

    把取值范围小的数据,转化成取值范围大的数据

    如int a = 10; double b = a;把a赋给b时自动提升了值的类型

  • 类型提升规则

    取值范围小的,和取值范围大的进行运算,小的会先提升为大的,再进行运算

    byte short char 三种类型的数据在运算到时候,会先直接提升为int,然后再继续运算

强制转换

  • 作用

    强制性把取值范围大的数据,赋值给取值范围小的变量

    如double b = 10.0; int a = b;就会报错,必须强转

  • 格式

    目标数据类型 变量名 = (目标数据类型)被强转的数据

    如int num = (int)13.14;

  • 结果可能出现精度错误

算术运算符的加法妙用

数字相加

  • 数字进行运算时,数据类型不一样不能计算,必须先转化成一样的才能进行运算(包括加减乘除)

字符串相加

当 + 操作中出现字符串时,这个 + 就会变成 字符串连接符 而不是算术运算符,会将前后两边的数据进行拼接,产生一个新的字符串

  • 如 ”123"+123 -> ”123123”

如果有连续的+操作,则从左到右逐个执行

  • 如 1+99+"长寿" -> “100长寿”

字符相加

当字符+字符or数字时,会先把字符通过 ASCII 码表转化成对应的整数再继续运算

  • ASCII码表 -> 65-90对应’A’-‘Z’,97-122对应’a’-‘z’

自增自减运算符

分类

减减

用法

如果是单独使用,无论放前面还是后面,结果都是+1或-1

如果参与运算,a++/a--表示先用a参与运算再对a+1/-1,即先用后加;反之先a+1/-1再用a参与运算,即先加后用

逻辑运算符

基本分类

分类

&,逻辑与 -> 并且,两边都为真,结果才为真

|,逻辑或 -> 或者,两边都为假,结果才为假

^,逻辑异或 -> 两边相同为false,两边不同为true

!,逻辑非 -> 取反

无论左边结果如何,右边表达式都要执行,没有短路效果

短路逻辑运算符

存在意义:短路实现代码逻辑的判断,提高效率

分类

&&,短路与,结果和&相同,但有短路效果

  • 比如a||b,a为真时不必判断b是否为真,直接确认结果为true

||,短路或,结果和|相同,但有短路效果

  • 比如a&&b,a为假时不必判断b是否为假,直接确认结果为false

短路效果

  • 当左边的表达式能确认结果,就不必判断右边的表达式

三元运算符

又称三元表达式

作用

作用

进行条件的判断, 然后根据判断的结果得到不同的内容

格式

格式

表达式 ? 表达式1 : 表达式2;

  • 根据关系表达式的结果选择内容,将其赋给变量

    如果表达式为true则选择表达式1

    如果表达式为false则选择表达式2

  • 例如 int max = a > b ? a : b;

    如果a>b,max=a

    如果b>a,max=b

作用:判断表达式结果,然后选择执行不同的代码块

执行流程

  1. 判断第一个表达式结果,为true则执行代码块1然后退出本if语句;为false则继续向下执行
  2. 判断第二个表达式结果,为true则执行代码块2然后退出本if语句;为false则继续向下执行
  3. 判断第n个表达式结果,为true则执行代码块n然后退出本if语句;为false则继续向下执行
  4. 所有表达式判断结果都不真,则执行最后的else部分语句体

  • 语句体中如果只有一行代码,可以省略{}

    2. 给变量a赋值10

  • 对一个布尔类型的变量进行表达式判断,不要用==,直接在括号中写入布尔类型变量名就好

作用:根据表达式结果,然后选择执行对应代码块

格式

JDK12之后,就有加强版switch语句;格式如下

关键字解释

  • switch 代表这部分switch选择语句
  • case 代表某个值部分的代码块
  • break 表示结束switch语句
  • default 表示所有值都不匹配时,执行的部分代码块

执行流程

  1. 进入switch语句后,计算表达式结果
  2. 从上到下,将表达式结果依次与值1-n进行比较,如果某一个值匹配了,就进入该部分语句体,然后执行代码;否则就接着向下判断
  3. 如果所有值都判断完不匹配,就执行default部分的语句体n+1,执行完退出switch语句

注意事项

  • default

    位置:位置可以放在switch语句中任意地方,但是建议放最后

    省略:可以不写default语句块,语法没问题,但是不建议省略(没备胎了)

  • case穿透

    产生原因:语句体中不写break语句

    导致结果:当匹配到某个语句体时,会从该语句体开始向下执行,除非遇到break语句或者遇到大括号}或者执行完毕,否则就会执行完下面的所有代码块

    使用场景:当多个case的语句体重复(比如1-3月春天xx),就可以用case穿透

  • switch新特性(JDK12->) 在上面有(即新写法)

  • switch选择语句与if判断语句的使用场景选择

    if判断语句一般用于对范围的判断

    switch选择语句一般用于对有限数据的列举选择

格式

  2. 执行条件判断语句,若为true执行循环体语句,然后执行3;若为false则退出for循环语句
  3. 执行完循环体语句就执行条件控制语句,更新条件变量

while循环语句

格式

执行流程

  1. 先执行初始化语句,然后进入while语句
  2. 进入while语句先执行条件判断语句,若为true则3,否则退出while语句
  3. 进入循环执行循环体语句,然后执行条件控制语句进行条件变量的更新
  4. 执行完条件控制语句就跳2

数组静态初始化

静态初始化

  • 初始化:在内存中,为数组开辟空间,并将数据存入数组的过程

  • 完整格式

    数据类型[] 数组名 = new 数据类型[]{元素1,元素2,元素3,...};

    例如:int[] arr = new int[]{1,2,3};

  • 简化格式

    数据类型[] 数组名 = {元素1,元素2,元素3,...};

    例如:int[] arr = {1,2,3};

  • 注意点

    数组一旦初始化,长度就固定了,不能再改变长度

数组的地址值

数组名代表数组容器在内存中的地址值

地址值的格式

[D@776ec8df

  • [表示当前是一个数组
  • D表示当前数组类型为double
    • I表示int
  • @表示一个间隔符号(固定格式)
  • 776ec8df数组真正的地址值(十六进制)

数组元素访问

格式

  • 数组名[索引];

    索引即数组的下标/角标,从0开始,逐个加1

可以利用索引,继续数组元素的访问与赋值

数组遍历

数组遍历

  • 将数组中所有的内容取出来,可以进行打印,求和,判断,赋值…
  • 数组遍历要记得↓
    • 遍历的最大索引是arr.length-1
    • 输入arr.fori自动生成有关数组arr的for循环

遍历的含义

指的是取出数据的过程,不要狭义地理解为遍历就是打印

动态初始化

作用

初始化时只指定数组的长度,由系统为数组分配默认初始值

  • 这样就不用先为每个元素赋值了

  • 默认初始值

    整数类型 -> 0

    小数类型 -> 0.0

    字符类型 -> ‘/u0000’ 空字符

    布尔类型 -> false

    引用数据类型 -> null

格式

格式

数据类型[] 数组名 = new 数据类型[数组长度];

比如int[] arr = new int[10];

动/静初始化区别

动态初始化和静态初始化的区别

动态初始化 -> 手动指定数组长度,由系统给出默认初始化值

  • 只知道元素个数,不确认具体数值时使用动态初始化

静态初始化 -> 手动指定数组元素,由系统计算给出数组长度

  • 明确了数据元素数据,直接使用静态初始化

数组的内存

Java内存分配

Java内存分配

  • 栈 -> 方法运行时使用的内存
  • 堆 -> 存储对象或数组,new出来的内容都存到堆里
  • 方法区 -> 存储运行的class文件
    • JDK8之后,取消方法区,新增元空间,将原来方法区的功能拆分到堆和元空间中了
  • 本地方法栈 -> JVM使用操作系统功能时使用,与开发无关
  • 寄存器 -> 给CPU使用,与开发无关

数组在内存的执行方式

  1. main方法进栈
  2. 在栈空间中创建arr变量,并在堆空间中开辟一块空间以初始化数组内容,然后栈中的arr变量存储这块堆空间的地址值
  3. 每次使用数组中的元素时,都根据arr存储的地址去对空间中查找目标内容
  4. main方法出栈,堆空间中的arr地址无变量指向,自动销毁
  • 若两个数组指向同一个地址(堆空间),则任意一个arr变量对地址中的数据进行修改,另一个arr变量获取的值也会发生改变

6’方法

介绍

方法(method)是程序中最小的执行单元,又称函数

作用

  • 提高代码复用性

    打包重复代码或特定功能代码,直接调用即可

  • 提高代码可维护性

方法的格式

方法就是把一些代码==打包==起来,需要的时候就==调用==

定义

将一些代码打包到一起的过程,称为方法的定义

格式

  • 如果没有返回值,返回值类型就写void ;return语句就写return;或者不写
  • 方法与方法之间是平级关系,不可嵌套定义

调用

手动调用使用某个方法的过程,称为方法的调用

格式

方法名(实参列表)

  • 如果有返回值,还要定义一个变量接收返回值
  • 调用时,实际参数的数量和类型必须与方法定义中的形参列表一一对应!

方法的重载

作用

  • 同一个方法名实现同种类型的功能

是什么

  • 同一个类中,定义了多个具有同种功能的同名方法,每个方法具有不同的参数类型和参数个数,这些同名的方法就构成重载关系

简单地说 -> 方法的重载就是同一个类中,方法名相同,参数不同,与返回值无关

  • 参数不同包括:个数不同,类型不同,顺序不同

方法的内存

方法调用的基本内存原理

  1. main方法进栈,栈空间为main方法开辟一块空间
  2. main方法中调用其他方法,其他方法进栈,栈空间为方法开辟一块空间
  3. 方法中的变量存储在方法的这块空间中
  4. 各个方法调用结束就出栈,其中的变量也会销毁
  5. 所有调用执行完毕,main方法出栈

基本数据类型&引用数据类型

本质区别

  • 基本数据类型 -> 变量存储的是数据值,数据存储在栈空间中
  • 引用数据类型 -> 变量存储的是地址值,数据存储在堆空间中(即new出来的)

方法传递基本数据类型的内存原理

比如说在main方法中调用一个change(int num)方法,传入num=100,方法中会重新对num赋值为200,现在想知道在main方法中执行完change方法后的num值是100还是200

  1. main方法进栈,定义并初始化num变量为100
  2. 调用change方法,change方法进栈
  3. change方法中定义num变量,传入了num=100,然后重新赋值num=200
  4. change方法出栈,num变量销毁
  5. main方法中num的结果还是100(因为4)
  • 所以,方法传递基本数据类型都是只传递数据,随着方法的出栈,数据也没了
    • 如果想要返回200这个数据值,就用return num;语句

传递基本数据类型时,传递的是真实的数据值,形参的改变不影响实参的值

方法传递引用数据类型的内存原理

同上一个案例,如果传入change方法的形参是arr[0],那么形参的改变也会影响实参

  1. main方法进栈,定义并初始化arr1[1]变量为100
  2. 调用change方法,change方法进栈
  3. change方法中定义arr2[]变量,传入了数组arr1地址值,然后重新赋值arr1[1]=200
  4. change方法出栈,arr2[]变量销毁
  5. main方法中arr1[1]的结果变成200
  • 所以,方法传递引用数据类型会传递地址,随着调用方法的出栈,地址中的已修改内容不会消失

传递引用数据类型时,传递的是地址值,形参的改变会影响实参的值
