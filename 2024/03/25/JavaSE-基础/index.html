字面量

是什么

字面量就是显示在屏幕上, 能直接看到的那些数据, 比如数字123, 字符abc等等

为什么

为什么要有字面量? => 规定书写数据的格式, 统一书写的规范

没有字面量, 程序员写在写代码中的数据的时候, 你写123, 我写123.0, 结果都是想表示数字123, 那岂不是就有各种各样的写法了, 所以需要一个规范, 写整数就是这么写, 写浮点数就是那样写..

怎么做

字面量的使用其实就是抓住 规范地书写数据 这一点

由于字面量分为 {整数, 小数, 字符串, 字符, 布尔值, 空值}, 那想要使用好字面量其实就是能书写正确即可, 要求如下

  • 整数类型 -> 不带小数点的数字,如666,-99
  • 小数类型 -> 带小数点的数字,如13.14,-3.22
  • 字符串类型 -> 用双引号””括起来的内容,如”abc”,”\t\n”
  • 字符类型 -> 用单引号’’括起来的单个内容,如’a’,’我’
  • 布尔类型 -> 布尔值,表示真假,只有true和false
  • 空类型 -> 一个特殊的值,空值null

除了上面的标准字面量, 程序中还有一些有特殊含义的字面量, 叫做 转义字符

  • 比如 \t 制表符, 它的功能就是 => 打印时,把前面字符串的长度补齐到8或8的整数倍.最少补1个空格,最多8个空格

类型转换

我们需要类型转换这个东西, 来对我们的数据的类型进行更改

隐式转换(自动类型提升)

  • 作用

    把取值范围小的数据,转化成取值范围大的数据

    如int a = 10; double b = a;把a赋给b时自动提升了值的类型

  • 类型提升规则

    取值范围小的,和取值范围大的进行运算,小的会先提升为大的,再进行运算

    byte short char 三种类型的数据在运算到时候,会先直接提升为int,然后再继续运算

显式转换(强制转换)

  • 作用

    强制性把取值范围大的数据,赋值给取值范围小的变量

    如double b = 10.0; int a = b;就会报错,必须强转

  • 格式

    目标数据类型 变量名 = (目标数据类型)被强转的数据

    如int num = (int)13.14;

  • 结果可能出现精度错误

取模%(取余) : 如10%2=0;2%5=2;

  • 浮点数不能取模

+ 号的进阶使用

=> 数字相加

  • 数字进行运算时,数据类型不一样不能计算,必须先转化成一样的才能进行运算(包括加减乘除)

=> 字符串相加

当 + 操作中出现字符串时,这个 + 就会变成 字符串连接符 而不是算术运算符,会将前后两边的数据进行拼接,产生一个新的字符串

  • 如 ”123"+123 -> ”123123”

如果有连续的+操作,则从左到右逐个执行

  • 如 1+99+"长寿" -> “100长寿”

=> 字符相加

当字符+字符or数字时,会先把字符通过 ASCII 码表转化成对应的整数再继续运算

  • ASCII码表 -> 65-90对应’A’-‘Z’,97-122对应’a’-‘z’

怎么用好自增运算符

自增运算符其实包括 自增 和 自减 运算

  • ++ -> 变量的值+1

  • -- -> 变量的值 -1

=> 基本使用

若只是变量单独的使用, 那么无论放前面还是放后面的效果都是一样的, 即+1/-1

如a++和++a; 本质都是对变量a+1, 没有区别

=> 进阶使用(用得也少)

如果自增/自减的变量还参与了运算, 那么++/–放前面和后面就有区别了

  • a++/a--表示先用a参与运算再对a+1/-1,即先用后加
  • 反之先a+1/-1再用a参与运算,即先加后用

=> 扩展赋值运算符的使用

+= ,由于原理相同,就以+=为例说明扩展赋值运算符,a += 10 <=> a = a + 10;

其实就是怎么用好”比较”这个运算

使用方法(左右两个独立表达式a,b)

  • == ,判断a和b的值是否相等,相等为true,不等为false
  • != ,判断a和b的值是否不相等,不相等为true,相等为false
  • > xxx

怎么用好逻辑运算符

逻辑运算符是用来处理语言逻辑 -> 与或非的运算符号

基本使用

&,逻辑与 -> 并且,两边都为真,结果才为真

|,逻辑或 -> 或者,两边都为假,结果才为假

^,逻辑异或 -> 两边相同为false,两边不同为true

!,逻辑非 -> 取反

利用这种方式想要得到运算的结果, 左右两边的表达式都要进行运算, 没有 短路效果

短路逻辑运算符使用

提高效率!!!

&&,短路与,结果和&相同,但有短路效果

  • 比如a||b,a为真时不必判断b是否为真,直接确认结果为true

||,短路或,结果和|相同,但有短路效果

  • 比如a&&b,a为假时不必判断b是否为假,直接确认结果为false

利用这种方式想要得到运算的结果, 只需要先对左边的表达式进行运算, 当左边的表达式能确定结果的时候, 就不算右边的表达式了, 具有 短路 的效果, 提高了开发效率

怎么用好三元运算符

三元运算符就是三元表达式, 即判断条件表达式的结果, 根据不同结果得到不同内容

使用方法

表达式 ? 表达式1 : 表达式2;

  • 根据关系表达式的结果选择内容,将其赋给变量

    如果表达式为true则选择表达式1

    如果表达式为false则选择表达式2

  • 例如 int max = a > b ? a : b;

    如果a>b,max=a

    如果b>a,max=b

背是很难的,但是现在可以记住一点 ↓

括号() 的优先级最高,可以利用()来选择自己想要提升运算优先级的部分

数组的内存

Java内存分配

  • 栈 -> 方法运行时使用的内存
  • 堆 -> 存储对象或数组,new出来的内容都存到堆里
  • 方法区 -> 存储运行的class文件
    • JDK8之后,取消方法区,新增元空间,将原来方法区的功能拆分到堆和元空间中了
  • 本地方法栈 -> JVM使用操作系统功能时使用,与开发无关
  • 寄存器 -> 给CPU使用,与开发无关

数组在内存的执行流程

  1. main方法进栈
  2. 在栈空间中创建arr变量,并在堆空间中开辟一块空间以初始化数组内容,然后栈中的arr变量存储这块堆空间的地址值
  3. 每次使用数组中的元素时,都根据arr存储的地址去对空间中查找目标内容
  4. main方法出栈,堆空间中的arr地址无变量指向,自动销毁
  • 若两个数组指向同一个地址(堆空间),则任意一个arr变量对地址中的数据进行修改,另一个arr变量获取的值也会发生改变

方法内存

方法调用的基本内存原理

  1. main方法进栈,栈空间为main方法开辟一块空间
  2. main方法中调用其他方法,其他方法进栈,栈空间为方法开辟一块空间
  3. 方法中的变量存储在方法的这块空间中
  4. 各个方法调用结束就出栈,其中的变量也会销毁
  5. 所有调用执行完毕,main方法出栈

方法传递基本数据类型参数的内存原理

比如说在main方法中调用一个change(int num)方法,传入num=100,方法中会重新对num赋值为200,现在想知道在main方法中执行完change方法后的num值是100还是200

  1. main方法进栈,定义并初始化num变量为100
  2. 调用change方法,change方法进栈
  3. change方法中定义num变量,传入了num=100,然后重新赋值num=200
  4. change方法出栈,num变量销毁
  5. main方法中num的结果还是100(因为4)
  • 所以,方法传递基本数据类型都是只传递数据,随着方法的出栈,数据也没了
    • 如果想要返回200这个数据值,就用return num;语句

传递基本数据类型时,传递的是真实的数据值,形参的改变不影响实参的值

方法传递引用数据类型参数的内存原理

同上一个案例,如果传入change方法的形参是arr[0],那么形参的改变也会影响实参

  1. main方法进栈,定义并初始化arr1[1]变量为100
  2. 调用change方法,change方法进栈
  3. change方法中定义arr2[]变量,传入了数组arr1地址值,然后重新赋值arr1[1]=200
  4. change方法出栈,arr2[]变量销毁
  5. main方法中arr1[1]的结果变成200
  • 所以,方法传递引用数据类型会传递地址,随着调用方法的出栈,地址中的已修改内容不会消失

传递引用数据类型时,传递的是地址值,形参的改变会影响实参的值

基本&引用数据类型本质区别

本质区别

基本数据类型 -> 变量存储的是数据值,数据存储在栈空间中

引用数据类型 -> 变量存储的是地址值,数据存储在堆空间中(即new出来的)

计算规则

公式

系数*基数的权次幂 相加

  • 基础 -> 当前进制数
  • 权 -> 从右往左, 依次为 0, 1, 2…

十进制转任意进制

除基取余法

开小灶-原码反码补码

原码

是什么

十进制数据的二进制表现形式, 就叫做一个数的原码, 最左边是符号位(0正1负)

比如 6这个数的二进制表现形式 0xxx0110, 就是6的原码, 且符号位为0, 表示整数

比特与字节

对于一个二进制数 (如110) 而言, 每一位数字都占用一个比特位bit

而8个比特就组成一个字节Byte

  • 字节是计算机中最小存储单元

在一个字节中, 能存储的最大数就是 0(代表正) 1111111 => 127

最小值就是 1(代表负) 1111111 => -127

为什么

为什么要有原码? => 将十进制数据用二进制表示

为什么在原码的表现形式下, 一个字节的最小值是 -127 ,而不是 -128 ?

=> 因为按照原码的定义, 确实一个字节表示的范围就是 [-127, 127], 只有后面按照补码的形式表示了, 才能取到 [-128, 127]

怎么做

原码计算

原码对正数进行计算是没问题的, 比如 101(5) + 10(2) 就是 111(7)

但是, 原码对负数进行计算, 就会有亿点点小问题

如 10xxx000 这个数表示 -0 , 在其基础上加1, 那就是 10xxx001, 即-1 !!! -0+1得到了-1,显然不合理

利用原码对负数进行加法运算, 本来想要的效果是往数轴右边走, 结果它往左边走

即 原码对负数进行运算, 实际的结果与预期相反

反码

是什么

为了解决原码不能进行负数运算的问题而出现的

正数的反码就是原码不变

负数的反码在原码的基础上, 符号位不变, 数值取反(0变1,1变0)

如 -56 => 10111000, 这是它的原码, 它的反码就是(符号位1不动, 后面的数值全部取反)

则为 11000111 , 然后对它进行 +5(101)的操作, 结果就是 11001100 这个反码, 然后再转为原码 10110011, 即负数 -51

计算结果正确~

为什么

为什么要有反码? => 解决负数的二进制计算问题

怎么做

反码计算

反码的计算就是, 现将原码转化为反码, 再执行对应的操作, 然后再转为原码就是最终的结果

转化为反码这一步, 就要遵循 正数反码=原码, 负数反码在原码的基础上符号位不动, 数值位0变1,1变0全取反

反码对于负数 -> 负数的计算是没有问题的

但是碰到 负数 -> 正数这种跨0情况时, 就会有一点问题

比如 -5 + 7 (10000101 + 00000111)

遇到负数参与运算, 首先转为反码, -5反码就是在原码基础上更改数值位, 即 11111010

然后和 00000111 相加, 结果是 00000001, 即正1, ??? -5+7应该是正2呀, 怎么会是正1呢?

所以, 利用反码对负数进行跨0运算, 会出现问题(吞掉了一个+1)

原因是 +0(00000000) 和 -0(10000000) 对于计算机而言是两个数, 而对于我们人是一个数, 导致了结果的偏移1位

如下示例

十进制 原码 反码
1 00000001 00000001
+0 00000000 00000000
-0 10000000 11111111
-1 10000001 11111110

从 -1 到 +1 之间, 有两个0~

为了解决这个问题, 就要用到补码

补码

是什么

为了解决反码计算负数跨0时的偏移问题而出现的

正数的原码=反码=补码

负数补码就是在负数反码的基础上, 错开了这一个偏移, 使得运算能够恢复正常

十进制 原码 反码 补码
1 00000001 00000001 00000001
+0 00000000 00000000 00000000
-0 10000000 11111111 00000000
-1 10000001 11111110 11111111
-2 10000010 11111101 11111110

从-0开始, 把上一位数的反码用于当前数的补码

扩大的特点

由于补码是在反码的基础上+1, 那么对于原码的最小值 11111111 而言, 转化为反码就是 10000000 ,再转为补码就是 10000001

就是说 -127这个数的补码还能再变小 即 10000000 这个特殊数, 它没有反码和原码

它就被用来表示 -128

由于计算机中的数字存储和计算都是以补码形式进行, 所以 一个字节Byte可以表示的范围为 [-128, 127]

为什么

为什么要有补码? => 解决反码跨0运算的+1偏移

怎么做

补码计算

补码的计算遵循 正数不动, 负数的补码就是在反码的基础上+1

如之前的例子 -5 + 7 (10000101 + 00000111)

此时先转为反码 11111010 + 00000111

再转为补码 11111011 + 00000111

结果就是 00000010 , 然后转回反码 00000010 ,转回原码还是00000010 即数字2

反码进行负数运算的跨0问题也解决了~

补码应用

计算机中的数字存储和运算, 都以补码形式进行

位运算符

逻辑与&

对二进制数进行运算, 运算规则 => 有0得0, 全1得1

如(int) 200 & 10

0000 0000 0000 0000 0000 0000 1100 1000

&

0000 0000 0000 0000 0000 0000 0000 1010

结果就是

0000 0000 0000 0000 0000 0000 0000 1000

即8

逻辑或|

对二进制数进行运算, 运算规则 => 有1得1, 全0得0

如(int) 200 | 10

0000 0000 0000 0000 0000 0000 1100 1000

&

0000 0000 0000 0000 0000 0000 0000 1010

结果就是

0000 0000 0000 0000 0000 0000 1100 1010

即202

左移运算<<

对二进制数进行运算, 运算规则 => 向左移动,低位补0

得到的效果 同乘2

如(int) 200 << 2

0000 0000 0000 0000 0000 0000 1100 1000

左移2位

0000 0000 0000 0000 0000 0011 0010 0000

即(512+256+32=800)

右移运算>>

对二进制数进行运算, 运算规则 => 向右移动,高位补0或1(负数补1)

得到的效果 同除2

如(int) 200 >> 2

0000 0000 0000 0000 0000 0000 1100 1000

右移2位

0000 0000 0000 0000 0000 0000 0011 0010

即(32+16+2=50)

无符号右移>>>

对二进制数进行运算, 运算规则 => 向右移动,高位必须补0

相比于右移>>, 其抛弃了正负符号的影响
