MySQL概述

数据库相关概念

数据库(DataBase -> DB)

  • 存储数据的仓库

数据库管理系统(DataBase Management System -> DBMS)

  • 操作和管理数据库的(系统)软件
  • 比如MySQL就是一个数据库管理系统

SQL(Structured Query Language,结构化查询语言)

  • 操作关系型数据库的编程语言,定义了操作关系型数据库的一套标准

一句话理解↓

用户通过SQL结构化查询语言来操作数据库管理系统DBMS,然后由DBMS对数据库DB中的数据进行管理

主流关系型数据库管理系统

关系型数据库

  • 关系型 -> 其实就是用二维表格来存储数据
  • RDBMS(关系型数据库) -> 建立在关系模型的基础上,由多张相互连接的二维表(就是表格)组成的数据库
  • 优点
    • 使用表结构存储数据 ,格式统一便于维护

常用DBMS

  • Oracle
  • MySQL
  • Microsoft SQL Server
  • PostgreSQL
  • ……

但是,无论哪一个数据库管理系统,都是以SQL为统一标准的

MySQL数据库

启动和停止

  • 方法一

    在cmd命令行窗口(win+r)中输入services.msc 进入本地服务界面,找到MySQL80即可手动选择开启/停止

  • 方法二

    在cmd命令行窗口中输入指令(记得开启管理员模式)

    • 启动

      net start mysql80

    • 停止

      net stop mysql80

这个MySQL80就是MySQL在Windows系统的默认服务名

客户端连接

  • 方法一

    使用MySQL提供的客户端命令行工具MySQL 8.0 Command Line Client ,然后输入密码就连接成功

  • 方法二

    使用Windows系统的命令行工具cmd,输入mysql [-h 127.0.0.1] [-p 3306] -u root -p,然后输入密码就连接成功

    想要在任意目录下的cmd中打开MySQL并连接客户端,就要配置其环境变量Path,新建一个C:\Program Files\MySQL\MySQL Server 8.0\bin\即配置成功

MySQL数据库数据模型

MySQL数据库的数据模型

  • 安装了MySQL之后,我们的电脑就成为MySQL数据库的服务器
  1. 客户端(用户电脑)可以连接MySQL数据库服务器,并使用SQL语句操作MySQL数据库管理系统
  2. 在MySQL数据库服务器内部
    1. 数据库管理系统DBMS可以管理多个数据库DB,数据库管理系统使用SQL语句管理数据库
    2. 一个数据库DB又可以管理多张表,数据库使用SQL语句管理二维表
    3. 每张表上又有多条记录(一行就是一条记录),二维表使用SQL语句管理记录
    4. 每条记录都对应着字段值(表格的标题单元格)

SQL(★)

通用语法

  • SQL语句可以单行/多行书写,以分号;结尾
  • SQL语句使用空格/缩进来增强语句可读性
  • MySQL中SQL语句不区分大小写
    • 关键字建议用大写
  • 注释
    • 单行注释 -> -- 注释内容 或 # 注释内容(MySQL特有)
    • 多行注释 -> /* 注释内容 */

SQL语句分类

  • DDL(Data Definition Language,数据定义语言)
    • 用来定义数据库对象(包括数据库,表,字段,索引等等)
  • DML(Data Manipulation Language,数据操作语言)
    • 用来对表中的数据进行增删改操作
  • DQL(Data Query Language,数据查询语言)
    • 用来查询数据库中表的记录
  • DCL(Data Control Language,数据控制语言)
    • 用来创建数据库用户,并控制数据库的访问权限

一句话理解SQL语句

DDL创建好数据库和库中的二维表后,由DML和DQL对表中数据执行增删改查操作,DCL则为用户和数据库设置权限

DDL

Data Definition Language,数据定义语言

用来定义数据库对象(包括数据库,表,记录,索引等

数据库操作

DDL-数据库操作

  • 查询

    查询所有数据库

    SHOW DATABASES;

    查询当前数据库

    SELECT DATABASE();

    这个语句一般忘记自己所在数据库时,查看当前所在数据库

  • 创建

    CREATE DATABASE [IF NOT EXISTS] 数据库名 [DEFAULT CHARSET 字符集] [COLLATE 排序规则];

    • 创建已有数据库会报错ERROR 1007 (HY000): Can't create database 'itcast'; database exists
    • 除非写上IF NOT EXISTS,表示不存在则创建,存在则不创建,且不会有语法错误
    • 创建数据库时指定字符集不要用utf8(因为每个字符3字节,mysql中有些字符需要4字节),而使用utf8mb4

  • 删除

    DROP DATABASE [IF EXISTS] 数据库名;

  • 使用

    USE 数据库名;

    即进入/切换某个选中数据库,进入后会有提示Database changed

表操作

DDL-表操作

  • 查询

    查询当前数据库的所有二维表

    SHOW TABLES;

    查询表结构

    DESC 表名;

    查询指定表的建表语句

    SHOW CREATE TABLE 表名;

  • 创建

    例子

    创建一个用户表 -> id name age gender

    使用DESC 表名;语句验证表的创建

    若出现几个字段,代表创建表成功

    使用SHOW CREATE TABLE 表名;详细地查看表的创建内容

  • 用于在创建表时,需要指定每个字段的类型,所以在下面的数据类型专门介绍mysql中数据的类型

  • 修改

    添加字段

    ALTER TABLE 表名 ADD 字段名 字段类型(长度) [COMMENT 注释] [约束];

    修改字段类型

    ALTER TABLE 表名 MODIFY 字段名 新数据类型(长度);

    修改字段名和字段类型

    ALTER TABLE 表名 CHANGE 旧字段名 新字段名 字段类型(长度) [COMMENT 注释] [约束];

    删除字段

    ALTER TABLE 表名 DROP 字段名;

    修改表名

    ALTER TABLE 表名 RENAME TO 新表名;

  • 删除

    删除表

    DROP TABLE [IF EXISTS] 表名;

    删除指定表,并重新创建该表 => 等价于删除表中记录数据

    TRUNCATE TABLE 表名;

    比如123.45这个小数

    • 标度D就是其小数位数2

字符串类型

类型 大小 描述
CHAR 0-255 Bytes 定长字符串
VARCHAR 0-65535 变长字符串
TINYBLOB 0-255 不超过255个字符的二进制数据
TINYTEXT 0-255 短文本字符串
BLOB 0-65535 二进制形式的长文本数据
TEXT 0-65535 长文本数据
MEDIUMBLOB 0-16777215 二进制形式的中等长度文本数据

LONGTEXT 0-4294967295 极大文本数据
  • BLOB描述二进制数据,TEXT描述文本数据
  • 定长字符串CHAR与变长字符串VARCHAR
    • 指定字符串最大长度(max_length)
    • 定长字符串CHAR无论存满没有,都占用max_length的字节
    • 变长字符串VARCHAR只占用当前存储内容的字节
    • CHAR性能高,VARCHAR性能低

案例-利用好数据类型创建员工信息表

DML

Data Manipulation Language,数据操作语言

用来对表中的数据进行增删改操作

添加/插入数据INSERT

DML-添加数据

  • 给指定字段添加数据

    INSERT INTO 表名 (字段1,字段2,...) VALUES (值1,值2,...);

  • 给全部字段添加数据

    INSERT INTO 表名 VALUES (值1,值2,...);

  • 批量添加数据

    指定字段添加多条记录

    INSERT INTO 表名 (字段1,字段2,...) VALUES (值1,值2,...),(值1,值2,...),(值1,值2,...);

    所有字段添加多条记录

    INSERT INTO 表名 VALUES (值1,值2,...),(值1,值2,...),(值1,值2,...);

  • ★★★注意事项★★★

    • 插入数据时,字段名和数据值一一对应
    • 字符串类型和日期时间类型的值要包含于引号中
    • 插入数据范围合法

修改/更新数据UPDATE

DML-修改数据

  • 修改数据

    UPDATE 表名 SET 字段名1=值1,字段名2=值2,...[WHERE 条件];

    若无WHERE 条件部分,则代表修改整张表的所有数据

删除数据DELETE

DML-删除数据

  • 删除数据

    DELETE FROM 表名 [WHERE 条件];

    若无WHERE 条件部分,则代表删除整张表的所有数据

    delete删除语句最小单位是一条记录(一行),不能删除某个字段的值(可以用update)

DQL

Data Query Language,数据查询语言)

用来查询数据库中表的记录

SELECT语句

QDL-查询数据

这是编写顺序~

基本查询

DQL-基本查询

  • 查询多个字段

    SELECT 字段1,字段2,字段3...FROM 表名;

    SELECT * FROM 表名;

    尽量不要写* 因为不直观/影响效率

  • 为查询返回的字段设置别名

    SELECT 字段1[AS 别名1],字段2[AS 别名2]...FROM 表名;

    AS可以省略

  • 为查询返回的记录去除重复记录

    SELECT DISTINCT 字段列表 FROM 表名;

条件查询

DQL-条件查询

  • 语法格式

    SELECT 字段列表 FROM 表名 WHERE 条件列表;

  • 条件

    比较运算符

    >, >=, <, <=, =, <>, != 大于,大于等于,小于,小于等于,等于,不等于,不等于

    BETWEEN...AND... 在某个范围之内(要求前小后大,否则无效)

    IN(匹配列表) 在in中的列表值多选一

    LIKE 占位符 模糊匹配(_匹配单个字符,%匹配任意个字符)

    IS NULL 是NULL

    逻辑运算符

    AND && 并且

    OR || 或者

    NOT ! 非,不是

聚合函数

DQL-聚合函数

配合分组查询使用

  • 介绍

    将一列数据作为一个整体,进行某个字段的纵向计算

  • 常见聚合函数

    count 统计数量

    max/min 最大值/最小值

    avg 平均值

    sum 求和

  • 语法格式

    SELECT 聚合函数(字段列表) FROM 表名;

  • 细节

    null值不参与聚合函数的运算

分组查询

DQL-分组查询

  • 语法格式

    SELECT 字段列表 FROM 表名 [WHERE 条件] GROUP BY 分组字段名 [HAVING 分组后过滤条件];

  • where和having的区别

    执行时机不同: where条件是对分组之前对结果进行过滤的,不满足where条件的不参与分组; 而having条件是分组之后对结果进行过滤

    判断条件不同: where条件不能对聚合函数进行判断; 而having条件可以对聚合函数进行判断

  • 细节

    执行顺序 : where > 聚合函数 > having

    分组之后,查询的字段一般为分组字段或聚合函数,查询其他字段没有意义

案例

  • 查询年龄小于45的员工,并根据工作地址分组,获取员工数量大于等于3的工作地址

  • 解析: 分组条件->工作地址 group by

    ​ 分组前过滤条件->年龄小于45 where

    ​ 分组后过滤条件->员工数量大于等于3 having

排序查询

DQL-排序查询

  • 语法

    SELECT 字段列表 FROM 表名 ORDER BY 字段1 排序方式1, 字段2 排序方式2;

  • 排序方式

    ASC 升序(默认值)

    DESC 降序

  • 多字段排序

    先按照第一个字段排序,如果第一个字段值相同,再按照第二个字段排序

分页查询

DQL-分页查询

  • 语法

    SELECT 字段列表 FROM 表名 LIMIT 起始索引, 查询记录数;

  • 起始索引

    从0开始,起始索引start_index=(查询页码 - 1)*每页显示记录数;

  • 细节

    分页查询是数据库的方言,不同数据库有不同语法实现,MySQL利用limit关键字

    若查询第一页数据,可以省略起始索引,直接写limit 查询记录数

DQL执行顺序

DQL-执行顺序

  1. FROM 先确定表
  2. WHERE 然后判断条件
  3. GROUP BY 然后分组
  4. SELECT 然后选择返回字段
  5. ORDER BY 然后排序
  6. LIMIT 最后分页

综合案例

  • 查询年龄为20,21,22,23的女员工信息
  • 查询性别为男,且年龄在20-40岁(包含边界)以内的姓名为三个字的员工
  • 统计员工表中,年龄小于60岁的男性员工和女性员工的人数
  • 查询所有年龄小于等于35岁员工的姓名和年龄,并对结果按年龄升序排序,如果年龄相同按入职时间降序排序
  • 查询性别为男,且年龄在20-40岁(包含边界)以内的前五个员工信息,对结果按年龄升序排序,年龄相同按入职时间升序排序

DCL

Data Control Language,数据控制语言

用来管理数据库用户,并控制数据库的访问权限

即每一个数据库有哪些用户可以访问 -> 管理数据库用户

每一个用户具有哪些访问权限(能访问哪些数据库) -> 数据库访问权限

用户管理

DCL-用户管理

  • 查询用户

    USE mysql;

    SELECT * FROM user;

  • 创建用户

    CREATE USER '用户名'@'主机名' IDENTIFIED BY '密码';

  • 修改用户密码

    ALTER USER '用户名'@'主机名' IDENTIFIED WITH mysql_native_password BY '新密码';

  • 删除用户

    DROP USER ‘用户名’@’主机名’;

  • 细节

    主机名->当前用户在哪一个主机上可以访问mysql

    • localhost表示当前主机
    • %表示任意主机

    用户名+主机名才能确定一个用户来访问mysql数据库

权限控制

DCL-权限控制

没有分配用户的访问权限时,新创建的用户只能访问information_schema这个系统数据库,只有为用户分配权限了,才能访问指定数据库

  • 常用权限

    ALL,ALL PRIVILEGES -> 所有权限

    SELECT -> 查询数据

    INSERT -> 插入数据

    UPDATE -> 修改数据

    DELETE -> 删除数据

    ALTER -> 修改表

    DROP -> 删除数据库/表/视图

    CREATE -> 创建数据库/表

  • 查询权限

    SHOW GRANTS FOR '用户名'@'主机名';

  • 授予权限

    GRANT 权限列表 ON 数据库名.表名 TO '用户名'@'主机名';

  • 撤销权限

    REVOKE 权限列表 ON 数据库名.表名 FROM '用户名'@'主机名';

  • 细节

    可以用通配符*代表”所有”,比如itcast.*就表示itcast数据库中的所有表

函数

一段可以被调用来实现某种功能的代码

分类

  • 字符串函数
  • 数值函数
  • 日期时间函数
  • 流程控制函数

字符串函数

  • 常用字符串函数

    CONCAT(S1,S2,...Sn) -> 将s1,s2,…sn拼接成一个字符串

    LOWER(str) -> 将字符串str转小写

    UPPER(str) -> 转大写

    LPAD(str,n,pad) -> 左填充,用pad串在str串的左边进行填充,使得总串达到n位长度

    RPAD(str,n,pad) -> 右填充,用pad串在str串的右边进行填充,使得总串达到n位长度

    TRIM -> 去掉字符串首尾的空格(有多少去多少)

    SUBSTRING(str,start,len) -> 返回str串从index索引开始,长度位len的子串

    • MySQL中索引index从1开始

  • 使用方式

    SELECT 函数(参数);

数值函数

  • 常用数值函数

    CEIL(x) -> 向上取整

    FLOOR(x) -> 向下取整

    MOD(x,y) -> 返回x%y的结果

    RAND() -> 生成返回0~1之内的随机数

    ROUND(x,y) -> 求参数x四舍五入的值,保留y位小数

日期时间函数

  • 常用日期时间函数

    CURDATE() -> 返回当前日期

    CURTIME() -> 返回当前时间

    NOW() -> 返回当前日期和时间

    YEAR(date) -> 获取指定date的年份

    MONTH(date) -> 获取指定date的月份

    DAY(date) -> 获取指定date的日期

    DATE_ADD(date,INTERVAL expr type) -> 返回一个日期/时间,加上一个时间间隔expr后的时间值

    • interval必写
    • expr就是表示间隔的年/月/日/时/分/秒的数值
    • type就是year/month/day/hour/minute/second

    DATEDIFF(date1,date2) -> 返回起始日期date1和结束日期date2之间的天数

    • 返回的是date1-date2的天数,可能会出现负数

流程控制函数

  • 作用

    在SQL语句中实现条件筛选

  • 常用流程控制函数

    IF(value,t,f) -> 如果value为true,返回t,否则f

    IFNULL(value1,value2) -> 如果value1不为空,返回value1,否则value2

    CASE WHEN [val] THEN [res1] ... ELSE [default] END -> 如果val1为true,返回res1,…,否则返回default默认值

    CASE [expr] WHEN [val1] THEN [res1] ... ELSE [default] END -> 如果expr的值为val1,返回res1,…,否则返回default默认值

  • 案例

    查询emp表的员工姓名和工作地址(如果地址是上海或背景,要展示一线城市,其他展示二线城市)

    统计班级学员成绩,展示规则 >=85 优秀,>=60 及格,否则 不及格

约束

限制表结构中存储的数据

介绍

  • 概念

    约束是作用于表中字段的规则,用来限制存储在表中的数据

  • 效果

    保证数据库中数据的正确,有效,完整性

  • 使用时机

    约束作用于表中字段,可以在创建表/修改表的时候为字段添加约束

  • 常用约束

    非空约束NOT NULL -> 限制该字段的数据不为空/null

    唯一约束UNIQUE -> 限制该字段的数据唯一不重复

    主键约束PRIMARY KEY -> 设置该字段为主键

    • 主键是一行数据的唯一标识,非空且唯一

    默认约束DEFAULT -> 保存数据时若未指定数据值,则采用默认值

    检查约束CHECK -> 保证字段值满足某一条件

    • 8.0.16版本后采用

    外键约束FOREIGN KEY -> 让两张表之间的数据建立连接

    自动增长AUTO_INCREMENT -> 自动增长,常与主键PRIMARY KEY结合使用

外键约束

外键用来让两张表的数据之间建立连接

语法

  • 添加外键

    创建表时添加外键约束

    修改表时添加外键约束

    ALTER TABLE 表名 ADD CONSTAINT 外键名称 FOREIGN KEY(外键字段名) REFERENCES 主表(主表列名);

  • 删除外键

    ALTER TABLE 表名 DROP FOREIGN KEY 外键名称;

  • 删除/更新行为

    常用行为

    NO ACTION -> 默认行为,当在父表中删除/更新对应记录时,首先检查该记录是否有对应外键,如果有则不允许删除/更新(与RESTRICT一致)

    RESTRICT -> 默认行为,当在父表中删除/更新对应记录时,首先检查该记录是否有对应外键,如果有则不允许删除/更新(与NO ACTION一致)

    CASCADE -> 级联,当在父表中删除/更新对应记录时,首先检查该记录是否有对应外键,如果有,则也删除/更新外键在子表中的记录

    SET NULL -> 当在父表中删除对应记录时,首先检查该记录是否有对应外键,如果有则设置子表中该外键值为null(要求子表中外键允许取null值)

    SET DEFAULT -> 父表有变更时,子表将外键列设置为一个默认值(INnodb不支持)

    行为语法

    ALTER TABLE 表名 ADD CONSTRAINT 外键名称 FOREIGN KEY (外键字段) REFERENCES 主表名(主表字段名) ON UPDATE 行为(如CASCADE) ON DELETE 行为(如CASCADE);

    • 主表即父表,从表即子表

笛卡尔积

笛卡尔积

  • 两个集合A,B的所有组合情况

    在多表查询的时候,就要消除无效的笛卡尔积

  • 例子

    一个集合内有a,b元素,另一个集合内有1,2,3,4元素,它们的笛卡尔积就是{(a,1),(a,2),(a,3),(a,4),(b,1),(b,2),(b,3),(b,4)}

案例

多表查询

  • 语法格式

    SELECT * FROM 表名1,表名2;

    然而,这样写就会出现”笛卡尔积”的问题,返回无效结果

    譬如 select * from emp,dept;

    就会得到这样的结果(6个员工*5个部门=>30条记录)

  • 消除无效笛卡尔积

    SELECT * FROM 表名1,表名2 WHERE 表1中主键字段=表2中外键字段;

    这样写了之后,就能将主表和从表的主键外键对应起来,不返回无效结果

    譬如select * from emp,dept where dept_id=dept.id;

    就会得到这样的结果(6个员工*5个部门并且主键外键值匹配)

分类

连接查询

  • 内连接 -> 相当于查询A,B表的交集数据
  • 外连接
    • 左外连接 -> 查询左表的所有数据 + 两张表的交集部分数据
    • 右外连接 -> 查询右表的所有数据 + 两张表的交集部分数据
  • 自连接 -> 当前表与自身的连接数据,自连接必须使用表别名
  • 联合查询 -> 返回多次查询的合并结果集

自连接

自连接查询,就是一张表连接自己

既可以是内连接查询,也可以是外连接查询

  • 语法

    自连接-内连接查询

    自连接-外连接(左)查询

    SELECT 字段列表 FROM 表A 别名A LEFT JOIN 表A 别名B ON 连接条件;

关键字 -> union, union all

union联合查询就是把多次查询的结果合并保存起来,形成一个新的查询结果集返回

  • 语法

子查询

介绍

  • SQL语句中嵌套SELECT语句的查询,就称为嵌套查询或子查询

    表现形式

    SELECT * FROM t1 WHERE column1 = (SELECT column1 FROM t2);

    子查询外部的语句可以是INSERT/UPDATE/DELETE/SELECT中的任何一个,常用SELECT语句

分类

根据子查询结果分类:

  • 标量子查询 -> 子查询结果为一个值
  • 列子查询 -> 子查询结果为一列
  • 行子查询 -> 子查询结果为一行
  • 表子查询 -> 子查询结果为多行多列

根据子查询位置分类:

  • WHERE之后
  • FROM之后
  • SELECT之后
