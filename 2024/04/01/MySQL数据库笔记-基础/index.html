MySQL概述

数据库相关概念

数据库(DataBase -> DB)

  • 存储数据的仓库

数据库管理系统(DataBase Management System -> DBMS)

  • 操作和管理数据库的(系统)软件
  • 比如MySQL就是一个数据库管理系统

SQL(Structured Query Language,结构化查询语言)

  • 操作关系型数据库的编程语言,定义了操作关系型数据库的一套标准

一句话理解↓

用户通过SQL结构化查询语言来操作数据库管理系统DBMS,然后由DBMS对数据库DB中的数据进行管理

主流关系型数据库管理系统

关系型数据库

  • 关系型 -> 其实就是用二维表格来存储数据
  • RDBMS(关系型数据库) -> 建立在关系模型的基础上,由多张相互连接的二维表(就是表格)组成的数据库
  • 优点
    • 使用表结构存储数据 ,格式统一便于维护

常用DBMS

  • Oracle
  • MySQL
  • Microsoft SQL Server
  • PostgreSQL
  • ……

但是,无论哪一个数据库管理系统,都是以SQL为统一标准的

MySQL数据库

启动和停止

  • 方法一

    在cmd命令行窗口(win+r)中输入services.msc 进入本地服务界面,找到MySQL80即可手动选择开启/停止

  • 方法二

    在cmd命令行窗口中输入指令(记得开启管理员模式)

    • 启动

      net start mysql80

    • 停止

      net stop mysql80

这个MySQL80就是MySQL在Windows系统的默认服务名

客户端连接

  • 方法一

    使用MySQL提供的客户端命令行工具MySQL 8.0 Command Line Client ,然后输入密码就连接成功

  • 方法二

    使用Windows系统的命令行工具cmd,输入mysql [-h 127.0.0.1] [-p 3306] -u root -p,然后输入密码就连接成功

    想要在任意目录下的cmd中打开MySQL并连接客户端,就要配置其环境变量Path,新建一个C:\Program Files\MySQL\MySQL Server 8.0\bin\即配置成功

MySQL数据库数据模型

MySQL数据库的数据模型

  • 安装了MySQL之后,我们的电脑就成为MySQL数据库的服务器
  1. 客户端(用户电脑)可以连接MySQL数据库服务器,并使用SQL语句操作MySQL数据库管理系统
  2. 在MySQL数据库服务器内部
    1. 数据库管理系统DBMS可以管理多个数据库DB,数据库管理系统使用SQL语句管理数据库
    2. 一个数据库DB又可以管理多张表,数据库使用SQL语句管理二维表
    3. 每张表上又有多条记录(一行就是一条记录),二维表使用SQL语句管理记录
    4. 每条记录都对应着字段值(表格的标题单元格)

SQL(★)

通用语法

  • SQL语句可以单行/多行书写,以分号;结尾
  • SQL语句使用空格/缩进来增强语句可读性
  • MySQL中SQL语句不区分大小写
    • 关键字建议用大写
  • 注释
    • 单行注释 -> -- 注释内容 或 # 注释内容(MySQL特有)
    • 多行注释 -> /* 注释内容 */

SQL语句分类

  • DDL(Data Definition Language,数据定义语言)
    • 用来定义数据库对象(包括数据库,表,字段,索引等等)
  • DML(Data Manipulation Language,数据操作语言)
    • 用来对表中的数据进行增删改操作
  • DQL(Data Query Language,数据查询语言)
    • 用来查询数据库中表的记录
  • DCL(Data Control Language,数据控制语言)
    • 用来创建数据库用户,并控制数据库的访问权限

一句话理解SQL语句

DDL创建好数据库和库中的二维表后,由DML和DQL对表中数据执行增删改查操作,DCL则为用户和数据库设置权限

DDL

Data Definition Language,数据定义语言

用来定义数据库对象(包括数据库,表,记录,索引等

数据库操作

DDL-数据库操作

  • 查询

    查询所有数据库

    SHOW DATABASES;

    查询当前数据库

    SELECT DATABASE();

    这个语句一般忘记自己所在数据库时,查看当前所在数据库

  • 创建

    CREATE DATABASE [IF NOT EXISTS] 数据库名 [DEFAULT CHARSET 字符集] [COLLATE 排序规则];

    • 创建已有数据库会报错ERROR 1007 (HY000): Can't create database 'itcast'; database exists
    • 除非写上IF NOT EXISTS,表示不存在则创建,存在则不创建,且不会有语法错误
    • 创建数据库时指定字符集不要用utf8(因为每个字符3字节,mysql中有些字符需要4字节),而使用utf8mb4

  • 删除

    DROP DATABASE [IF EXISTS] 数据库名;

  • 使用

    USE 数据库名;

    即进入/切换某个选中数据库,进入后会有提示Database changed

表操作

DDL-表操作

  • 查询

    查询当前数据库的所有二维表

    SHOW TABLES;

    查询表结构

    DESC 表名;

    查询指定表的建表语句

    SHOW CREATE TABLE 表名;

  • 创建

    例子

    创建一个用户表 -> id name age gender

    使用DESC 表名;语句验证表的创建

    若出现几个字段,代表创建表成功

    使用SHOW CREATE TABLE 表名;详细地查看表的创建内容

  • 用于在创建表时,需要指定每个字段的类型,所以在下面的数据类型专门介绍mysql中数据的类型

  • 修改

    添加字段

    ALTER TABLE 表名 ADD 字段名 字段类型(长度) [COMMENT 注释] [约束];

    修改字段类型

    ALTER TABLE 表名 MODIFY 字段名 新数据类型(长度);

    修改字段名和字段类型

    ALTER TABLE 表名 CHANGE 旧字段名 新字段名 字段类型(长度) [COMMENT 注释] [约束];

    删除字段

    ALTER TABLE 表名 DROP 字段名;

    修改表名

    ALTER TABLE 表名 RENAME TO 新表名;

  • 删除

    删除表

    DROP TABLE [IF EXISTS] 表名;

    删除指定表,并重新创建该表 => 等价于删除表中记录数据

    TRUNCATE TABLE 表名;

MySQL中的数据类型主要分为

  • 数值类型
  • 字符串类型

数值类型

类型 大小 有符号(SIGNED)范围 无符号(UNSIGNED)范围 描述
TINYINT 1Byte (-128,127) (0,255) 小整数值
SMALLINT 2Bytes (-32768,32767) (0,65535) 大整数值
MEDIUMINT 3 (-8388608,8388607) (0,16777215) 大整数值
INT/INTEGER 4 (-2147483648,2147483647) (0,4294967295) 大整数值
BIGINT 8 (-2^63^,2^63^-1) (0,2^64^-1) 极大整数值
FLOAT 4 大 大 单精度浮点数值
DOUBLE 8 超大 超大 双精度浮点数值
DECIMAL 依赖于M(精度)和D(标度)的值 依赖于M(精度)和D(标度)的值 小数值(精确定点数)

  • DECIMAL依赖精度/标度的值

    比如123.45这个小数

    • 精度M就是其数字长度5
    • 标度D就是其小数位数2

  • 如何声明数据类型采用无符号UNSIGNED范围

    age tinyint unsigned

  • 声明float/double/decimal数据类型的时候, 要加上 (数字长度,小数位个数) 这个后缀

    比如float(4,1)这个小数, 就表示一共4位长度, 且小数位1位, 那么整数位最多3位

字符串类型

类型 大小 描述
CHAR 0-255 Bytes 定长字符串
VARCHAR 0-65535 变长字符串
TINYBLOB 0-255 不超过255个字符的二进制数据
TINYTEXT 0-255 短文本字符串
BLOB 0-65535 二进制形式的长文本数据
TEXT 0-65535 长文本数据
MEDIUMBLOB 0-16777215 二进制形式的中等长度文本数据
MEDIUMTEXT 0-16777215 中等长度文本数据
LONGBLOB 0-4294967295 二进制形式的极大文本数据
LONGTEXT 0-4294967295 极大文本数据
  • BLOB描述二进制数据,TEXT描述文本数据
  • 定长字符串CHAR与变长字符串VARCHAR
    • 指定字符串最大长度(max_length)
    • 定长字符串CHAR无论存满没有,都占用max_length的字节
    • 变长字符串VARCHAR只占用当前存储内容的字节
    • CHAR性能高,VARCHAR性能低

案例-利用好数据类型创建员工信息表

DML

Data Manipulation Language,数据操作语言

用来对表中的数据进行增删改操作

添加/插入数据INSERT

DML-添加数据

  • 给指定字段添加数据

    INSERT INTO 表名 (字段1,字段2,...) VALUES (值1,值2,...);

  • 给全部字段添加数据

    INSERT INTO 表名 VALUES (值1,值2,...);

  • 批量添加数据

    指定字段添加多条记录

    INSERT INTO 表名 (字段1,字段2,...) VALUES (值1,值2,...),(值1,值2,...),(值1,值2,...);

    所有字段添加多条记录

    INSERT INTO 表名 VALUES (值1,值2,...),(值1,值2,...),(值1,值2,...);

  • ★★★注意事项★★★

    • 插入数据时,字段名和数据值一一对应
    • 字符串类型和日期时间类型的值要包含于引号中
    • 插入数据范围合法

修改/更新数据UPDATE

DML-修改数据

  • 修改数据

    UPDATE 表名 SET 字段名1=值1,字段名2=值2,...[WHERE 条件];

    若无WHERE 条件部分,则代表修改整张表的所有数据

删除数据DELETE

DML-删除数据

  • 删除数据

    DELETE FROM 表名 [WHERE 条件];

    若无WHERE 条件部分,则代表删除整张表的所有数据

    delete删除语句最小单位是一条记录(一行),不能删除某个字段的值(可以用update)

DQL

Data Query Language,数据查询语言)

用来查询数据库中表的记录

SELECT语句

QDL-查询数据

这是编写顺序~

基本查询

DQL-基本查询

  • 查询多个字段

    SELECT 字段1,字段2,字段3...FROM 表名;

    SELECT * FROM 表名;

    尽量不要写* 因为不直观/影响效率

  • 为查询返回的字段设置别名

    SELECT 字段1[AS 别名1],字段2[AS 别名2]...FROM 表名;

    AS可以省略

  • 为查询返回的记录去除重复记录

    SELECT DISTINCT 字段列表 FROM 表名;

条件查询

DQL-条件查询

  • 语法格式

    SELECT 字段列表 FROM 表名 WHERE 条件列表;

  • 条件

    比较运算符

    >, >=, <, <=, =, <>, != 大于,大于等于,小于,小于等于,等于,不等于,不等于

    BETWEEN...AND... 在某个范围之内(要求前小后大,否则无效)

    IN(匹配列表) 在in中的列表值多选一

    LIKE 占位符 模糊匹配(_匹配单个字符,%匹配任意个字符)

    IS NULL 是NULL

    逻辑运算符

    AND && 并且

    OR || 或者

    NOT ! 非,不是

聚合函数

DQL-聚合函数

配合分组查询使用

  • 介绍

    将一列数据作为一个整体,进行某个字段的纵向计算

  • 常见聚合函数

    count 统计数量

    max/min 最大值/最小值

    avg 平均值

    sum 求和

  • 语法格式

    SELECT 聚合函数(字段列表) FROM 表名;

  • 细节

    null值不参与聚合函数的运算

分组查询

DQL-分组查询

  • 语法格式

    SELECT 字段列表 FROM 表名 [WHERE 条件] GROUP BY 分组字段名 [HAVING 分组后过滤条件];

  • where和having的区别

    执行时机不同: where条件是对分组之前对结果进行过滤的,不满足where条件的不参与分组; 而having条件是分组之后对结果进行过滤

    判断条件不同: where条件不能对聚合函数进行判断; 而having条件可以对聚合函数进行判断

  • 细节

    执行顺序 : where > 聚合函数 > having

    分组之后,查询的字段一般为分组字段或聚合函数,查询其他字段没有意义

案例

  • 查询年龄小于45的员工,并根据工作地址分组,获取员工数量大于等于3的工作地址

  • 解析: 分组条件->工作地址 group by

    ​ 分组前过滤条件->年龄小于45 where

    ​ 分组后过滤条件->员工数量大于等于3 having

排序查询

DQL-排序查询

  • 语法

    SELECT 字段列表 FROM 表名 ORDER BY 字段1 排序方式1, 字段2 排序方式2;

  • 排序方式

    ASC 升序(默认值)

    DESC 降序

  • 多字段排序

    先按照第一个字段排序,如果第一个字段值相同,再按照第二个字段排序

分页查询

DQL-分页查询

  • 语法

    SELECT 字段列表 FROM 表名 LIMIT 起始索引, 查询记录数;

  • 起始索引

    从0开始,起始索引start_index=(查询页码 - 1)*每页显示记录数;

  • 细节

    分页查询是数据库的方言,不同数据库有不同语法实现,MySQL利用limit关键字

    若查询第一页数据,可以省略起始索引,直接写limit 查询记录数

DQL执行顺序

DQL-执行顺序

  1. FROM 先确定表
  2. WHERE 然后判断条件
  3. GROUP BY 然后分组
  4. SELECT 然后选择返回字段
  5. ORDER BY 然后排序
  6. LIMIT 最后分页

综合案例

  • 查询年龄为20,21,22,23的女员工信息
  • 查询性别为男,且年龄在20-40岁(包含边界)以内的姓名为三个字的员工
  • 统计员工表中,年龄小于60岁的男性员工和女性员工的人数
  • 查询所有年龄小于等于35岁员工的姓名和年龄,并对结果按年龄升序排序,如果年龄相同按入职时间降序排序
  • 查询性别为男,且年龄在20-40岁(包含边界)以内的前五个员工信息,对结果按年龄升序排序,年龄相同按入职时间升序排序

DCL

Data Control Language,数据控制语言

用来管理数据库用户,并控制数据库的访问权限

即每一个数据库有哪些用户可以访问 -> 管理数据库用户

每一个用户具有哪些访问权限(能访问哪些数据库) -> 数据库访问权限

用户管理

DCL-用户管理

  • 查询用户

    USE mysql;

    SELECT * FROM user;

  • 创建用户

    CREATE USER '用户名'@'主机名' IDENTIFIED BY '密码';

  • 修改用户密码

    ALTER USER '用户名'@'主机名' IDENTIFIED WITH mysql_native_password BY '新密码';

  • 删除用户

    DROP USER ‘用户名’@’主机名’;

  • 细节

    主机名->当前用户在哪一个主机上可以访问mysql

    • localhost表示当前主机
    • %表示任意主机

    用户名+主机名才能确定一个用户来访问mysql数据库

权限控制

DCL-权限控制

没有分配用户的访问权限时,新创建的用户只能访问information_schema这个系统数据库,只有为用户分配权限了,才能访问指定数据库

  • 常用权限

    ALL,ALL PRIVILEGES -> 所有权限

    SELECT -> 查询数据

    INSERT -> 插入数据

    UPDATE -> 修改数据

    DELETE -> 删除数据

    ALTER -> 修改表

    DROP -> 删除数据库/表/视图

    CREATE -> 创建数据库/表

  • 查询权限

    SHOW GRANTS FOR '用户名'@'主机名';

  • 授予权限

    GRANT 权限列表 ON 数据库名.表名 TO '用户名'@'主机名';

  • 撤销权限

    REVOKE 权限列表 ON 数据库名.表名 FROM '用户名'@'主机名';

  • 细节

    可以用通配符*代表”所有”,比如itcast.*就表示itcast数据库中的所有表

函数

一段可以被调用来实现某种功能的代码

分类

  • 字符串函数
  • 数值函数
  • 日期时间函数
  • 流程控制函数

字符串函数

  • 常用字符串函数

    CONCAT(S1,S2,...Sn) -> 将s1,s2,…sn拼接成一个字符串

    LOWER(str) -> 将字符串str转小写

    UPPER(str) -> 转大写

    LPAD(str,n,pad) -> 左填充,用pad串在str串的左边进行填充,使得总串达到n位长度

    RPAD(str,n,pad) -> 右填充,用pad串在str串的右边进行填充,使得总串达到n位长度

    TRIM -> 去掉字符串首尾的空格(有多少去多少)

    SUBSTRING(str,start,len) -> 返回str串从index索引开始,长度位len的子串

    • MySQL中索引index从1开始

  • 使用方式

    SELECT 函数(参数);

数值函数

  • 常用数值函数

    CEIL(x) -> 向上取整

    FLOOR(x) -> 向下取整

    MOD(x,y) -> 返回x%y的结果

    RAND() -> 生成返回0~1之内的随机数

    ROUND(x,y) -> 求参数x四舍五入的值,保留y位小数

日期时间函数

  • 常用日期时间函数

    CURDATE() -> 返回当前日期

    CURTIME() -> 返回当前时间

    NOW() -> 返回当前日期和时间

    YEAR(date) -> 获取指定date的年份

    MONTH(date) -> 获取指定date的月份

    DAY(date) -> 获取指定date的日期

    DATE_ADD(date,INTERVAL expr type) -> 返回一个日期/时间,加上一个时间间隔expr后的时间值

    • interval必写
    • expr就是表示间隔的年/月/日/时/分/秒的数值
    • type就是year/month/day/hour/minute/second

    DATEDIFF(date1,date2) -> 返回起始日期date1和结束日期date2之间的天数

    • 返回的是date1-date2的天数,可能会出现负数

流程控制函数

  • 作用

    在SQL语句中实现条件筛选

  • 常用流程控制函数

  • 删除/更新行为

    常用行为

    NO ACTION -> 默认行为,当在父表中删除/更新对应记录时,首先检查该记录是否有对应外键,如果有则不允许删除/更新(与RESTRICT一致)

    RESTRICT -> 默认行为,当在父表中删除/更新对应记录时,首先检查该记录是否有对应外键,如果有则不允许删除/更新(与NO ACTION一致)

    CASCADE -> 级联,当在父表中删除/更新对应记录时,首先检查该记录是否有对应外键,如果有,则也删除/更新外键在子表中的记录

    SET NULL -> 当在父表中删除对应记录时,首先检查该记录是否有对应外键,如果有则设置子表中该外键值为null(要求子表中外键允许取null值)

    SET DEFAULT -> 父表有变更时,子表将外键列设置为一个默认值(INnodb不支持)

    行为语法

    ALTER TABLE 表名 ADD CONSTRAINT 外键名称 FOREIGN KEY (外键字段) REFERENCES 主表名(主表字段名) ON UPDATE 行为(如CASCADE) ON DELETE 行为(如CASCADE);

事务

介绍

事务是一组操作的集合,是一个不可再分割的工作单位

事务会把所有操作作为一个整体向系统提交请求或撤销操作请求,所以这些操作要么同时成功,要么同时失败

例子

  • 张三给李四银行卡转账

    操作1 查询张三账户余额有没有1000

    操作2 张三账户余额-1000

    操作3 李四账户余额+1000

  • 这三个操作作为一个整体事务,要么成功要么失败,不能再分割操作

事务的操作流程

  • 开启事务
  • 进行事务中的若干操作
  • 如果某个操作出现异常,就回滚事务(撤销之前临时进行的操作)
  • 如果所有操作执行成功,提交事务,完成操作

在MySQL中的事务是自动提交的,当执行一条DML语句时,MySQL立即隐式提交事务

事务操作

控制事务有两种方式

第一种事务控制方式

查看/设置事务提交方式

  • 查看事务提交方式

    SELECT @@autocommint;

    若为1->自动提交;为0->手动提交

  • 设置事务提交方式(设置为手动)

    SET @@autocommit = 0;

提交事务

  • COMMIT;

回滚事务

  • ROLLBACK;

第一种事务的控制方式是修改事务提交方式为手动,仅当手动提交时才会完成事务

第二种事务控制方式

不修改事务的提交方式

开启事务

  • START TRANSACTION or BEGIN;

提交事务

  • COMMIT;

回滚事务

  • ROLLBACK;

第二种事务的控制方式是开启事务后必须提交事务,否则也不完成事务

事务四大特性

四大特性(ACID)

  • 原子性(Atomicity) : 事务是不可分割的最小操作单元,要么all成功,要么all失败
  • 一致性(Consitency) : 事务完成时,必须使所有数据都保持一致状态
  • 隔离性(Isolation) : 数据库系统提供的隔离机制,保证事务在不受外部并发影响的独立环境下运行
  • 持久性(Durability) : 事务一旦提交或回滚,它对数据库中的数据改变都是永久的

并发事务问题

多个并发事务在执行过程中出现的问题

常见三个问题

  • 脏读

    一个事务读取到另一个事务还没有提交的数据

    比如A事务中对id字段从1修改为2,B事务同时访问数据库表中的id字段,获取到了A事务还没有提交的id字段修改值2,就是脏读

  • 不可重复读

    一个事务先后读取同一条记录,但两次读取的数据不同

    比如A事务先查询了数据库表,获得了id值为1的数据,同时B事务对该id值进行了修改为2的操作并且提交事务了,然后A事务再次查询id字段值,获得了值为2的结果,导致两次查询结果不一致,这就是不可重复读

  • 幻读

    一个事务按照条件查询数据时,没有对应的数据行,但是又在插入数据时,发现这行数据已经存在,好像出现了”幻影”

    比如A事务根据条件来查询数据库表时,没有查询到对应结果,同时B事务向数据库提交了插入id=1的操作,然后A事务想要插入id=1的时候发现数据库表中已存在数据结果,在解决了不可重复读问题的基础上,A事务再次查询还是没有对应结果,这就是幻读

事务隔离级别

分类

  • Read uncommitted 读,未提交
  • Read committed 读,已提交
  • Repeatable Read MySQL默认隔离级别,可重复读
  • Serializable 串行化

可解决的问题(×代表解决)

隔离级别 脏读 不可重复读 幻读
Read uncommitted √ √ √
Read committed × √ √
Repeatable Read × × √
Serializable × × ×
  • 从上到下 隔离级别(数据安全性)↑↑, 性能↓↓

查看事务隔离级别

  • SELECT @@TRANSACTION_ISOLATION;

设置事务隔离级别

  • SET [SESSION|GLOBAL] TRANSACTION ISOLATION LEVEL {READ UNCOMMITTED | READ COMMITTED | REPEATABLE READ | SERIALIZABLE};

​ 同时开两个cmd窗口执行SQL语句,以模拟两个并发事务,以下简称事务A和事务B

    B中执行commit;

    A中执行select * from account; 得到张三余额1000李四2000

    验证了read commited会出现不可重复读问题

  • 幻读问题

    设置事务隔离级别为repeatable read

    先验证该隔离级别已经解决了不可重复读的问题

    A中开启事务,执行select * from account; 得到张三余额1000李四余额2000

    B中开启事务,执行update account set money = money + 1000 where name = '张三';和commit; 修改张三余额为2000并提交

    A中再执行select * from account; 得到张三余额1000李四2000

    A中执行commit; 得到张三余额2000李四2000

    A中张三余额未提交时一直是1000未变动,不受B中已修改为2000的影响

    验证了repeatable read解决了不可重复读的问题

    再验证该隔离级别会出现幻读问题

    A中开启事务,执行select * from account where id = 3; 返回结果为空

    B中开启事务,执行insert into account (id,name,money) values(3,'王五',2000); 和commit; 插入id为3的记录并提交

    A中也执行insert into account (id,name,money) values(3,'大王五',2000); 提示id=3记录已存在

    A中再执行select * from account where id = 3; 返回结果还是为空

    A中最后执行commit; 然后执行select * from account where id = 3; 返回有此结果

    验证了repeatable read会出现幻读问题

  • Serializable解决所有问题

    串行化在执行并发事务操作时,一次只允许一个事务来操作,其他事务操作不被允许

    验证Serializable解决了幻读的问题

    A中开启事务,执行select * from account where id = 4; 返回结果为空

    B中开启事务,执行insert into account (id,name,money) values (4,'panda',2000); 然而进度被阻塞了

    只有A中执行完 insert into account (id,name,money) values (4,'panda',2000); 和 commit; 后在B中返回id=4记录已存在,阻止插入

    B中执行 insert into account (id,name,money) values (4,'panda',2000); 返回结果已存在
