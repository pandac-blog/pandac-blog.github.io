面向对象是什么

面向对象编程, 即OOP(Object-Oriented Programming)

它的核心就是, 程序员所写的代码和代码执行的作用, 都是面向于对象的

什么叫面向于对象? 面向 -> 拿/取, 对象 -> 能干活的东西

对象就可以理解为工具, 然后我们要解决问题的时候, 就要用这个工具来解决对应的问题, 这就可以类比为 : 我们人类解决问题是面向工具的, 那么对于程序而言, 其解决问题就是面向对象的 即 面向对象就是拿能干活的东西来解决问题

面向对象三大特性是什么

封装 -> 封装指的是将数据和操作数据的方法捆绑在一起的机制，以隐藏对象的内部状态并限制对其访问

继承 -> 继承指的是一个类可以使用另一个类的属性和方法，并可以重写或添加新的功能

多态 -> 多态指的是同一个方法可以在不同的对象上产生不同的行为

封装

封装介绍

封装是什么

封装是设计类/对象的规则

它告诉我们 如何正确地设计一个类/对象的属性和方法

目的是将对象的属性和属性对应的行为捆绑在一起, 限制类外内容对其的访问

封装的原则就是 对象代表什么, 就要封装对应的数据, 并提供数据相关的行为

  • 比如,人这个类,和圆这个类,当有一个”画圆”的方法时,应该设计到圆这个类中,因为画圆这个行为属于圆这个类/对象
  • 再比如, 张三砍李四导致李四嗝屁, 嗝屁这个行为就是李四这个对象的行为, 只是张三调用了李四嗝屁的行为, 导致了李四的嗝屁, 本质上还是李四自己的嗝屁行为导致了李四的嗝屁
封装的核心思想是什么

对象代表什么, 就要封装对应的数据, 并且提供数据相关的行为

  • 所以, 封装其实主要针对类中的成员方法和成员属性

而类中的构造方法是为了初始化对象的, 也可以算作封装的一个内容

但是类中的代码块和内部类这两块内容并不属于封装的内容, 而笔者还是将其归类到封装模块下~

目的是使大家能够以封装继承多态的宏观视角理解面向对象, 不要拘泥于细节

创建对象的内存分析

本篇主要理解, 创建一个对象时的内存分析

Java内存分配

Java虚拟机JVM会和其他应用程序(比如说QQ, 网页)一样, 占用内存中的部分空间, 以执行程序

而为了更好地管理这块内存空间, JVM将其占用的内存分为5个部分↓

  • 栈

    方法被调用时, 需要进栈执行, 其中的变量也会进行初始化, 等方法体执行完毕后, 方法就要出栈, 其中声明的临时存储在栈空间中的变量也会销毁

  • 堆

    所有被new出来的内容, 都存储在堆空间中, 这些被new出来的内容会有一个地址值表示其在堆空间中的位置

  • 方法区(元空间)

    前世今生

    • jdk7以前, 方法区和堆是连在一起的, 而这种设计并不好

      jdk8之后, 新增了元空间, 取消方法区, 将元空间和堆分开, 将原来方法区中的部分功能分到元空间, 还有一部分分到堆空间

      但是在篇内存分析的讲解中, 笔者还是将元空间命名为原本的方法区, 便于理解

    功能

    • 运行一个类的时候, 这个类的字节码文件就会加载到方法区中, 临时存储

  • 本地方法栈

  • 寄存器

一个对象的内存分析

比如说创建一个Student类的对象student Student student = new Student();

在JVM分配的内存中, 至少要执行以下5步

  1. 在方法区中加载Student.class字节码文件, 临时存储类中的方法

  2. 方法入栈, 在栈空间中声明局部变量student

    这个student变量存储的内容其实就是将要赋给它的在堆空间中的地址值

  3. 在堆空间中开辟一块空间, 用来存储student对象中的内容

    它在堆空间中的位置就是它的地址值

  4. 初始化

    默认/显式/构造方法初始化都是为student对象中的成员变量进行赋值/初始化

    默认初始化, 就是基本数据类型赋初始值(0, 0.0, ‘\u0000’, false), 引用数据类型赋null值

    显式初始化, 就是在类中定义成员变量的时候直接赋值了, 比如private int age =20; , 那么默认初始值就会被其覆盖

    构造方法初始化, 就是创建类的对象时, 调用有参构造将参数赋给对象的成员变量

  5. 将堆空间中的地址值赋值给栈空间中的局部变量student

以实际例子来说明, 比如现在有一个Student类(①), 然后要在Test类(②)的main方法中, 执行 Student s = new Student(); 这条创建对象语句, 那就要走5步流程

  1. 方法区加载Test类的字节码文件Test.class, 临时存储其中的main方法

  2. main方法入栈, (↑↑想要声明Student的变量s, 就要先在方法区中再加载Student类的字节码文件Student.class, 临时存储类的成员属性和方法), 然后在栈空间中声明并存储临时变量s

  3. 然后在堆空间中开辟空间分配给s对象, 这块内存里面就存储s对象中的成员属性的值, 和成员方法的地址(指向方法区中存储的方法)

  4. 对s对象进行初始化

    此案例中采用默认初始化

  5. 将堆空间中s对象地址值赋给栈空间中s变量

两个对象的内存分析

在上述案例创建s对象的基础上, 再创建一个s2对象

那么还是走5步流程

  1. 加载Test.class, Studnet.class字节码文件, 临时存储方法
  2. main方法入栈, 声明临时变量s和s2
  3. 堆空间中开辟两块空间, 分给s对象和s2对象
  4. 对这两个对象初始化
  5. 将s和s2对象在堆空间中的地址值赋给栈空间中的s和s2变量

两个对象变量指向同一块堆空间地址

即Student s1 = new Student(); 然后 Student s2 = s1;

在第二创建对象语句时, 就会将s1在堆空间中的地址值赋给s2变量, 这样两个对象变量指向同一块空间, 任何一方执行的操作都会影响堆空间中存储的数据, 间接影响到另一个对象

this关键字

就近原则是什么

谁离我近, 我就用谁

比如有两个同名变量number, 一个在全局作用, 一个在函数体内部作用, 那当代码执行到函数体内部的时候, 如果没有特殊声明, 那肯定用的是函数体中的number变量, 因为它离得近

由于就近原则的特点, 在类中书写方法涉及到对属性的操作时, 就要注意是否混淆了全局变量(即成员属性)和局部变量(即方法中定义的变量)

this关键字是什么

this关键字本质就是当前对象的地址值(即方法的调用者的地址值)

作用是用this指向的变量表示类中的成员属性变量, 而不是传入的局部变量, 可以 区分成员变量和局部变量

在非静态方法中, 都 有一个隐藏的this形参 , 表示是当前这个对象在调用此方法

比如 public void show(Student this){} 就有一个默认隐藏this形参, 方便对象的调用

this关键字内存分析

一个类中的this关键字的本质是 方法调用者的地址值

笔者个人理解, 其实可以把this关键字(存储某个对象地址值)看成一个类中的成员变量, 哪个对象调用类中的方法, 这个this存储的就是这个对象的地址值

成员&局部变量的区别

成员变量 -> 类中的属性(在方法体之外)

​ 比如Student类中定义的int age; String name;都是成员变量

局部变量 -> 方法中的变量(在方法体之内)

​ 比如setAge方法中定义的形参(setAge(int age)中的age), 这就是一个局部变量, 方法被调用后, 执行完毕(出栈)后就会被销毁

二者区别如下(不用记)

区别 成员变量 局部变量
类中位置不同 类中, 方法体之外 类中, 方法体之内
初始化值不同 有默认初始化值 没有默认初值, 需要自行赋值
内存位置不同 堆内存空间 栈内存空间
生命周期不同 随着对象的创建而创建, 随着对象的销毁而销毁 随着方法的调用而产生, 随着方法的出栈(运行完毕)而销毁
作用域不同 整个类中有效 当前方法体中有效

static静态修饰符

static静态修饰符是什么

可以将 static 理解为 => 其 表示了类中各对象共享的内容

被static修饰的成员变量和成员方法, 就会被类中所有对象共享, 而且还具有一些特点(详细如下)

静态变量和静态方法是什么

被static修饰的成员变量,就叫做静态变量

特点

  • 静态变量被这个类的所有对象共享
  • 静态变量不属于某个对象,而属于这个类
  • 随着类的加载而加载,优先于对象存在

被static修饰的成员方法,就叫做静态方法

特点

  • 静态方法被这个类的所有对象共享

  • 多用在测试类和工具类中,JavaBean类中很少用

    因为利用static修饰的成员方法 - 静态方法, 调用很方便

static的内存分析

比如现在利用Student学生类创建两个学生对象stu1和stu2

学生类中声明了一个静态变量bossName

然后内存中的执行流程大概就是

  1. Student.class字节码文件加载到方法区, 临时存储方法

  2. main方法进栈, 初始化静态变量, 并创建两个学生变量

    Student.bossName=”xx”;

    在这里, 就会对学生类中的共享内容 - bossName进行赋值, 会将bossName的值 存储到堆空间中的静态区(静态存储位置) , 其存储着这个类的所有静态变量 => 静态变量随着类的加载而加载, 优先于对象出现

    而这个类中的静态方法存储在方法区中

  3. 同时在堆空间中, 开辟两块空间分给这两个学生对象

    每个学生对象都会指向堆空间中静态区里面的静态变量, 这样就可以用学生对象的名称调用静态变量

  4. 初始化堆空间中的对象内容

  5. 将学生对象地址赋给学生栈中学生变量

  6. main出栈, 销毁学生变量, 而无变量指向学生对象, 其也自动销毁

静态的访问权限

静态方法只能访问静态变量/方法,即静态内容

如何理解? -> 字面理解就是, 一个静态方法不能访问非静态的内容, 也很好说明, 静态内容是类中所有对象共享的, 而非静态内容是类的每个对象都有的, 如果静态方法能够访问非静态内容, 那它访问的是哪个对象的非静态变量/方法呢?

显然不合理, 所以静态方法只能访问静态内容, 这些静态内容属于类, 类的对象也有资格访问, 反过来就不行


非静态方法可以访问静态/非静态变量&方法,即所有内容

在上面结论及说明的基础上, 可以很好地说明这一结论, 由于非静态方法是类创建的对象中的内容, 那么非静态方法是肯定能访问非静态内容的, 因为它们都属于这个对象, 使用这个对象的this存储地址就能正确调用成员方法和变量

而静态内容是属于类的, 类的对象也可以访问(只是不推荐), 所以非静态方法既可以访问非静态内容, 也可以访问静态内容


静态方法中没有this关键字

this关键字是调用此方法者的地址值, 而只有对应类的对象才能调用类中的方法, 那这些对象是有资格拥有this关键字来记录每个对象的地址值的

而对于静态方法的拥有者 -> 类而言, 都是直接以类名来调用静态方法, 没有必要存储一个this关键字来记录对象的地址值(根本就没有/不需要对象), 所以静态方法中书写this关键字是错误的

总结

  • 静态方法只能访问静态

    因为我们同归属于类

  • 非静态可以访问所有

    因为对象有资格访问类的静态内容

  • 静态方法没有this关键字

    因为静态方法属于类, 而不是对象, 不需要this存储对象地址值

重新认识main方法

  • public -> 访问权限够大

  • static -> 归属于类, 用类名访问

    由于main是静态static修饰的, 所以main方法同一类中的所有方法也用static修饰, 才可调用

  • void -> 无返回值

  • main -> 一个通用名称, 虽然不是关键字, 但是JVM还是要识别到main才进行程序

  • String[] args -> 以前用于接收键盘录入数据, 现在没有了, 不过为了jdk向下兼容还是保留了

比如说学生类, 现在3年2班的同学有若干位, 然后我想执行为学生类的属性boss(班主任)赋值, 那他们的班主任应该是同一个人, 那直接 Student.bossName=”张三”; 即可, 不用对每一个学生对象单独赋班主任的值(如 studentN.bossName=”张三”; N=0,1,2…)

继承的关系体系是什么

继承的父子关系 既可以表示为直接父子关系, 也可以表示为间接父子关系

直接父类和间接父类的概念

  • 直接父类, 子类直接继承的父类

    比如说 Student extends Person, Person就是Student的直接父类

  • 间接父类, 子类继承的父类所继承的父类

    比如说 Student extends Person, MiddleStudent extends Student, Student就是MiddleStudent的直接父类, 而Person就是MiddleStudent的间接父类

但是, 无论是直接父类还是是间接父类, 父类都有一个尽头叫做 Object类 , 其为Java中所有类的祖宗

  • 即Java中所有类都直接或间接继承Object类

并且, 只要是子类继承了父类, 无论是直接还是间接父类, 子类都能使用从父类中继承的内容

误区驳回

  • 误区1 -> 父类中私有的内容, 子类就无法继承
  • 误区2 -> 父类中非私有的内人, 子类就可以继承

这两个关于继承的观点都是错误的!

父类中的成员内容↓

  • 构造方法
  • 成员变量
  • 成员方法

这三个内容就是子类可能会从父类中继承的东西

结论表 - 子类能继承父类中的哪些内容

父类成员 访问权限 子类能否继承
构造方法 非私有 不能
私有private 不能
成员变量 非私有 能
私有private 能
成员方法 虚方法 能
(主要看是不是虚方法) 非虚方法 不能

父类中的构造方法无论权限如何, 子类都无法继承

  • 因为子类的类名是newClass, 而父类的类名是oldClass, 父类的构造方法名也是oldClass(), 假如子类继承了父类的构造方法, 那就会导致 子类类名 ≠ 子类构造方法名 违背了构造方法的定义
  • 所以在子类中, 必须重新编写构造方法

父类中的成员变量无论权限如何, 子类都能继承下来, 能不能用另说

  • 父类中非私有的成员变量子类可以直接继承使用

  • 父类中私有的成员变量子类可以继承, 但也无法使用

    若实在想要使用父类中的私有成员变量, 可以调用父类的set/get方法

  • 子类继承父类的成员变量, 并不会把父类的变量抢过来用, 而是拷贝一个变量存储在子类中

父类中的成员方法必须是虚方法子类才能继承, 否则子类不会继承

  • 父类中的方法满足了非private/static/final时, 即该方法是父类中的虚方法时, 才会被子类继承到虚方法表中
虚方法表

虚方法表/虚方法是什么

JVM将类中可能会用到的方法抽取出来, 存放到虚方法表中

哪些方法(虚方法)能被存到虚方法表?

  • 非private
  • 非static
  • 非final

满足以上三点的成员方法, 就会保存到该类的虚方法表中, 然后继承下去

虚方法表的继承特点

  • 继承时, 后者在前者基础上, 继承父类的虚方法表, 并添加自己类的虚方法进去, 这样保证每个类都会有自己的虚方法表, 用到哪些成员方法直接在虚方法表中查找调用

虚方法表的存储位置

  • 随着类的字节码文件加载, 同所有成员方法一起存储在方法区(元空间)中
  • 而所有类都直接/间接地继承Object类, 所以在加载这些类的时候, 也会加载Object类, 包括 其包含的5个虚方法 , 存储在方法区

为什么要有虚方法表

提高性能!提高性能!提高性能!

如果没有虚方法表, 那一个多层继承了20层的子类, 想要调用前20层的父类中的方法, 那就要一层一层找上去, 效率太低

有了虚方法表后, 每个子类都有属于自己的虚方法表来存储虚方法, 想要使用哪个方法直接调用即可, 效率upupup!

方法的重写

方法重写是什么

子类继承父类的成员方法后, 又声明一个相同的成员方法 , 就称子类的这个成员方法是重写的方法

方法重写这个概念的前提

  • 基于继承

    没有子类继承父类, 就没有所谓方法重写这个概念

方法重写的本质

  • 覆盖虚方法表中的方法

    子类中重写的方法覆盖了从父类虚方法表中继承而来的方法

    比如说A类里面有fun1和fun2两个虚方法, B类继承A类, 然后重写fun2-B方法

    那么B类的虚方法表中, 就会有A-fun1这个方法和B-fun2这个方法, 也就是B-fun2覆盖了A-fun2这个继承过来的方法

为什么要重写类中的方法

当 父类的方法不能满足子类的需求时, 子类就要重写方法

如何重写类中的方法

  1. 加上@Override注解

    Override注解能够检验子类重写的方法语法是否正确, 建议重写的方法都要加上Override注解

  2. 更改方法体内容

方法重写的要求

  • 方法的名称, 形参列表与父类中方法一致

  • 子类重写父类方法, 子类方法的访问权限必须 >= 父类

    默认(空着不写) < protected < public

  • 子类重写父类方法, 子类方法的返回值类型必须 <= 父类

    不过2, 3两点都比较少用到

  • 只有父类中的虚方法子类才可以重写

总而言之, 子类重写父类方法, 尽量保持一致即可~

抽象类和抽象方法

抽象类和抽象方法是什么?

先记住一点 -> 抽象类/抽象方法和方法的重写是绑定在一起的

  • 当涉及子类对父类中继承的方法进行重写时, 就要用abstract关键字修饰父类中的那个方法, 表示强制子类要重写该方法

  • 此时, 父类中的该方法就是抽象方法, 而包含抽象方法的类, 就叫抽象类

抽象方法是一种声明而不是实现具体功能的方法, 具体的实现由其子类提供(即在父类中不可确定具体方法体)

抽象类是一种特殊的类, 它包含了抽象方法

为什么要有抽象方法?

=> 利用abstract关键字修饰父类中的方法, 强制继承该方法的子类要重写这个方法, 起到一个规范的作用

为什么要有抽象类?

=> 用来表示这个类中有抽象方法, 需要被重写, 而且这个类不能实例化

利用抽象方法和抽象类, 可以强制子类按照某种格式编写代码, 统一规范!!!

继承时构造方法的访问特点

父类中的构造方法不会被子类继承

子类中的所有构造方法默认先访问父类中的无参构造, 再执行自己

  • 为什么会有先访问父类无参构造这一步?

    因为在子类对象的堆空间内存中, 一部分存储属于自己的成员变量, 还有一部分存储从父类继承而来的成员变量

    那么, 当我们创建一个子类对象(即子类执行一次构造方法)的时候, 就要先为堆空间中”从父类继承而来的对象”进行初始化, 有可能之后会用到父类中的数据

    这也就是执行子类构造方法前, 先访问父类无参构造的意义, 要将继承而来的成员变量赋默认值, 方便子类对象后面的操作

    为什么不能访问父类的有参构造呢? 因为顶级父类Object没有成员变量, 就没有 有参的构造方法

  • 怎么执行先访问父类无参构造这一步?

    在子类的构造方法中, 默认第一句 super(); , 表示其父类的构造方法

    如果还想调用父类的有参构造, 那就必须在子类构造方法第一行手撕 super(参数列表); 这句代码

    super()必须第一行!!!

一个小问题…

  • 学生管理系统中有很多人员, 比如老师, 学生, 系统管理员这三种身份, 当我们注册创建一个账号的时候, 就会碰到这种问题:

    public static void register() {} , 这个注册方法中, 传入的参数到底是{Student s, Teacher t, Administrator a}中的哪一个?

    如果说是Student s, 那其他两个类怎么创建身份? 以此类推, 选择这三个中的任何一个作为参数, 都不合理……

以前的解决方案

  • 既然一个register方法传入哪一个参数都不行, 那我就写三个register方法, 每个方法传入s, t, a这三个对象中的每一个

    但是这种方法并不好, 为什么? => 在之后添加一个辅导员的身份的时候, 还要再写一个register方法, 没有程序追求的通用性

    所以我们追求一种通用的解决方案, 要求一个register方法既能接收s,t,a这三种角色, 还要能接收未来可能创建的其他角色

现在的解决方案

  • 在传入参数处, 写上这几个角色的共同父类Person的对象 => public static void register(Person p) {}

    此时, 假如传入”Teacher t”这个对象, 那么就是以父类变量接收子类对象 Person p = new Teacher(); , 符合多态的要求

更重要的一点

  • 利用多态的特性, 不仅仅可以使用父类变量接收子类对象, 还能根据传入对象的不同, 调用属于不同对象的成员方法


总结, 为什么要有多态? => 在类与类的继承关系之上, 帮助我们进行父子类调用的统一

  • 编译时，确定访问的是引用类型（左边）中的成员变量。编译器会检查引用类型中是否存在该成员变量，如果存在则编译通过，否则报错。
  • 运行时，实际访问的也是引用类型（左边）中的成员变量。因为成员变量没有多态的特性，所以无论实际对象的类型是什么，访问的都是引用类型中的成员变量。

所以，在多态调用成员变量时，无论是编译时还是运行时，都会看左边（引用类型），访问的都是引用类型中的成员变量。


对于调用成员方法再次总结一下：

  • 编译时，确定要调用的是引用类型（左边）中的方法。编译器会检查引用类型中是否存在该方法的声明，如果存在则编译通过，否则报错。
  • 运行时，实际执行的是对象的类型（右边）中的方法。这是多态的核心特性，即使编译时选择的是引用类型中的方法，但实际执行的却是对象类型中的方法版本。

因此，在多态调用成员方法时，编译时看的是左边（引用类型），运行时执行的是右边（实际对象类型）中的方法。

接口是什么

接口就是一种规则, 实现这个接口的类(即实现类)就可以拥有某种功能

所以 理解好extends, Interface 和 implements三个关键字 之间的区别就可以理解好接口和类

接口和抽象类的区别是什么

类(指JavaBean类)是描述事物的内容的组合体

抽象类则是在继承基础上, 为了更方便地规范父子类抽象方法的关系的类

  • 所以抽象类是建立在父子类继承关系上的
  • 类与类之间继承(extends)

接口则是一种规则, 它允许类去实现这个规则以获得某种功能

  • 所以接口是建立在对于某种规则(行为)的获得上的
  • 接口与类之间实现(implements)

但是, 二者的相同点就是, 父类/接口中都是 只有声明方法, 而没有具体实现

即都是抽象方法, 需要继承的类/实现的类去重写方法体

接口和类的关系

接口和类的实现关系

  • 可以单实现, 也可以多实现

    public class 类名 implements 接口1, 接口2 {}

    当实现类实现的多个接口中, 有同名方法时, 重写方法时只会重写一次, 表示既重写了接口1中的抽象方法, 也重写了接口2中的抽象方法

  • 实现类可以在继承一个类的同时, 实现多个接口

    public class 类名 extends 父类 implements 接口1, 接口2 {}

类和类的继承关系

  • 可以单继承, 不能多继承, 可以多层继承

    public class 子类 extends 父类 {}

接口和接口的继承关系

  • 可以单继承, 也可以多继承

    如果 实现类实现的是继承体系最下面的子接口, 就要重写该子接口父类的所有抽象方法

    例如下方例子, Inter3接口继承了Inter1和Inter2接口, 在InterImpl实现类实现Inter3接口时, 必须重写Inter3包括Inter1和Inter2中的抽象方法

为什么要有接口

接口可以为继承体系中的类, 添加某种规则, 使得类可以实现接口中的抽象方法, 拥有某种功能, 而这个功能是其他类所不具有的

比如说一个简单的继承体系, Animal动物类下子类青蛙Frog, 狗Dog, 兔子Rabbit; 各种继承了吃饭, 喝水等行为…

但是, 此时想要描述青蛙和狗会游泳 => 即类中包含游泳行为的时候, 就不好描述了

我们不能在父类Animal中直接定义游泳行为, 因为子类中还有不会游泳的(Rabbit); 当然也不要在子类中直接定义游泳行为, 因为这样没有规范, 可能在青蛙类中定义是swimming, 狗类中定义的就是swim

所以我们 需要接口, 来表示一种规则, 而实现这个规则就可以拥有某种功能, 且这个功能是具有差异性的

字符串拼接的底层原理

字符串对象str拼接分为两类

  • 无变量参与拼接
  • 有变量参与拼接

无变量参与拼接

例如 String str = “a” + “b” + “c"; , 右边没有变量参与拼接, 都是字符串字面量参与拼接

此时会触发字符串的优化机制, 编译时就已经是最终的结果

=> 底层会将分开的若干字符串看作一个字符串对象

即 str=”a” + “b” + “c”; <=> str = “abc”;

有变量参与拼接

例如 String str = str1 + “PandaC”; , 右边有变量参与字符串对象的拼接

JDK8以前使用StringBuilder类

  • 先在串池中加载 str1(“a”) 和 “PandaC”

StringBuilder整体设计

  • StringBuilder创建一个sb对象时, 会自动创建一个容量(存储的最大长度, 而长度表示已经存储的长度)为16的字节数组

  • double sqrt(double a) -> 获取正平方根

每个 Java 应用程序都有一个 Runtime 类实例，使应用程序能够与其运行的环境相连接

即 => 表示当前虚拟机的运行环境 (不是工具类, 且无法使用构造方法创建对象, 可以通过getRuntime()获取当前运行实例)

对象克隆方式详解

第一种方式 - 浅克隆/拷贝

特点 => 直接拷贝

比如说将A对象的属性数据克隆/拷贝到B对象中, 那么就会将原对象(即A对象)中的属性值直接拷贝给B对象的属性

  • 基本数据类型 => 拷贝数据值

    比如说A对象中的int age = 20属性, 拷贝给B对象时就是age = 20, 直接将数据值拷贝过来

  • 引用数据类型 => 拷贝地址值

    比如说A对象中的int[] num = {1,2,3}, 地址值0x0011, 拷贝给B对象时就是int[] num = {1,2,3}(B) -> 0x0011, 直接将地址值拷贝过来

    浅克隆/拷贝 有一个问题, 那就是拷贝引用数据类型的时候, 两个对象的属性指向的是同一块地址 , 任何一方进行数据的改变, 另一个对象得到的数据就会发生变化

第二种方式 - 深克隆/拷贝

特点 => 间接拷贝

比如说将A对象的属性数据克隆/拷贝到B对象中, 那么就会将原对象(即A对象)中的属性值间接拷贝给B对象的属性

  • 基本数据类型 => 拷贝数据值

    和浅拷贝基本数据类型的方式一致, 也是直接拷贝数据值过去

  • 引用数据类型 => 创建新内存地址, 将原对象的引用类型数据值赋给这块新内存, 而不是直接赋地址值

    比如说A对象中的int[] num = {1,2,3}, 地址值0x0011, 拷贝给B对象时就是int[] num = {1,2,3}(B) -> 0x0022, 然后将0x0011中的数据值拷贝给0x0022这块地址, 然后让num(B)指向0x0022这块地址, 并不是直接将地址值拷贝过来

    深克隆/拷贝 就解决了浅克隆引用数据类型的共用地址而造成的共享数据问题 , 任何一方进行数据的改变, 另一个对象的数据都不会发生变化

    但是! 对于String类创建的非new对象, 其有一个小例外, 因为此String对象通过 赋值=运算 创建, 故其存储于 StringTable串池 中, 创建一个新对象时, 就会复用原来的字符串地址, 对于这种String对象的拷贝, 拷贝的地址值也是串池中的相同地址 ~

总结 - 对象的克隆/拷贝方式

  • 浅克隆 -> 无论对象的属性是基本数据类型还是引用数据类型, 直接拷贝过来(数据值 or 地址值)
  • 深克隆 -> 对象的属性若是基本数据类型, 直接拷贝; 若是引用数据类型, 则会创建新空间存储相同数据(如果是非new创建的字符串会复用串池地址)

BigInteger

是什么

不可变的任意精度的整数

即 => 可以操作大整数的类

☆注意★ => 对象一旦创建, 内部的值不可发生改变

如何使用

构造方法(修饰符都是public)

  • BigInteger(int num, Random rnd) -> 获取随机大整数, 范围[0, 2^num^-1]

  • BigInteger(String val) -> 获取指定大的整数

  • BigInteger(String val, int radix) -> 获取指定进制的大整数


还有一个静态成员方法获取BigInteger对象

  • 对象一旦创建, 内部的值不会发生改变, 每次计算都会产生一个新的对象

BigDecimal

计算机中的小数

计算机中数据都以补码二进制形式进行运算

  • 小数(浮点数)也不例外, 例如一个69.875的十进制小数, 其包括的内容就包括十进制的整数和十进制的小数部分

    将两个部分转为二进制就是 (0100 0101) 和 (111)

    (0100 0101) => 69转为二进制的原码, (111) => 0.875转为二进制的原码(1 * 2^-1 + 1 * 2^-2 + 1 * 2^-3)

但是呢, 由于小数位的数字难以表示(0.9要用45位01码表示, 0.226要用55位01吗表示), 而float(小数有23bit)和double(小数有52bit)占用的字节数又是有限的

  • 所以对于 超出的部分, 只能舍弃, 这就导致了数据的不精确

为了解决小数表示的不精确(金融, 精密仪器等行业), Java提出了BigDecimal类

  • 用来进行小数的精确运算
  • 还用来表示很大的小数

是什么

不可变的、任意精度的有符号十进制数

即 => 可以操作大浮点数的类

  • 可以进行小数的精确运算
  • 可以进行超大小数的表示

如何使用

构造方法

  • public BigDecimal(String val) -> 将字符串对象转为BigDecimal对象, 而且数据值保证了精确性, 建议使用

还有一个静态成员方法获取BigDecimal对象

  • public static valueOf(long/double val) -> 调用静态方法, 传入long或double数值, 创建BigDecimal对象

是什么

将各种规则拼接在一起, 组成而成的一个字符串, 就是正则表达式Regex

作用

  • 校验字符串是否符合自定义的规则
  • 在一段文本中查找满足要求的内容

如何使用(校验字符串是否符合规则)

使用格式

  • 用待校验字符串 调用matches(“Regex”)方法 , Regex即为字符串形式的正则表达式内容

    例如上面”先爽一下”的 qq.matches("[1-9]\\d{5,19}") ,就是用qq这个待校验字符串调用matches方法, 传递参数就是正则表达式字符串

语法规则

  • 字符类(只匹配一个字符)

    [abc] -> 只能是a, b或c

    [^abc] -> 除了a, b, c之外的任何字符

    [a-zA-Z] -> 从a到z, 或从A到Z

    [a-d[m-p]] -> 从a到d, 或从m到p

    [a-z&&[def]] -> 从a到z, 和d,e,f的交集 => 即d, e, f

    • 表示两个范围的交集一定要用两个&符号 -> &&
    • 如果写成一个&符号 -> & ,就表示简简单单的一个&字符

    [a-z&&[^bc]] -> 从a到z, 和非b,c的交集 => 等价于[ad-z], ad到z

    [a-z&&[^m-p]] -> 从a到z, 和非m到p的交集 => 等价于[a-lq-z], a - lq - z(忽略mnop)

正则表达式使用小结

符号 含义 举例
[] 包括的内容仅出现一次 [0-9], [a-zA-Z0-9]
() 分组 a(bc)+
^ 取反 [^abc]
&& 交集, 不可写一个& [a-z&&m-p]
` ` 并集(写在括号外, 括号内连写自动表示并集)
. 任意字符 不包括\n, 回车符号
\ 转义字符 \\d
\\d 数字0-9 \\d+
\\D 非数字0-9 \\D+
\\s 空白字符 [\t\n\x0B\f\r]
\\S 非空白字符 [^\\s]
\\w 单词字符 [a-zA-Z_0-9]
\\W 非单词字符 [^\\w]
? 0次或1次 \\d?
* 0次或多次 \\d*, (abc) *
+ 1次或多次 \\d+, (abc)+
{} 具体次数 a{7}, \\d{7,19}
(?i) 忽略后面字符的大小写 (?i)abc
a((?)b)c 只忽略b的大小写 a((?)b)c
