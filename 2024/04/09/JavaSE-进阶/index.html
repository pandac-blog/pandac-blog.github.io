类与对象

Java中,必须先定义类,才能创建对象

  • 类 -> 对象共同特征的描述,就像一个设计图

类的注意事项

测试类

  • 以前在main()方法里执行各种功能,包含着main方法的类就是测试类,用来执行功能

JavaBean类

  • 比如Student,Phone..都是JavaBean类,用来描述一类事物

封装

什么是封装

​ => 封装告诉我们,该如何正确地设计类/对象的属性和方法

  • 对象代表什么,就得封装对应的数据,并提供数据对应的行为
    • 比如,人这个类,和圆这个类,当有一个”画圆”的方法时,应该设计到圆这个类中,因为画圆这个行为属于圆这个类/对象

封装的好处/作用

  • 程序员能正确设计类/对象
  • 程序员可以直接调用已经写好的类/对象中的方法来解决问题,不用再手撕代码了,降低学习成本

private关键字

是什么?

  • 一个权限修饰符,表示私有

有什么用?

  • 可以修饰成员(变量/方法),被修饰的成员只能在本类中才可以被访问

由于private限制了访问权限,所以初始化变量/调用变量时需要使用set/get方法

set/get方法

setXyz(参数列表)表示对调用者的成员变量进行赋值/初始化

getXyz()表示获取调用者的成员变量

this关键字

一个问题

在getXyz(参数列表)方法中,以Student类为例,其中有name属性

name = name 这个时候本意是把传入的name赋值给成员变量name,此时遵循就近原则,谁离的近就用谁,但是容易产生歧义

解决方法 -> 使用this关键字

this关键字

this关键字

是什么

  • 表示某个对象的地址值

有什么用

  • 在对对象的变量赋值时,用this.调用该变量表示这是成员变量而不是传入的局部变量
  • 区分成员变量和局部变量

构造方法

介绍

构造方法是什么

  • 也称构造器,构造函数

作用

  • 创建对象时,由JVM自动调用,对成员变量进行初始化

书写格式

格式

书写要求

  • 构造方法名与类名完全相同
  • 构造方法没有返回值类型,连void空类型都没有
  • 构造方法没有具体返回值(不可以写return语句)

执行时机

  • 创建对象时有虚拟机自动调用,不可手动调用
  • 每创建一个对象,就会调用一次构造方法

空参构造

  • 如果类中没有写构造方法,JVM会自动添加一个空参构造;如果定义了构造方法,JVM就不会自动添加空参构造了
  • 空参构造和有参构造都必须写上
  • 空参构造和有参构造就是构造方法的重载

标准JavaBean类

介绍

  1. 类名需要见名知意,驼峰命名
  2. 成员变量用private修饰
  3. 提供至少两个构造方法
    • 空参构造
    • 带全部参数的有参构造
  4. 成员方法
    • 提供每一个成员变量的setXyz()/getXyz()方法
    • 如果有其他行为,也要加上

快速生成方法

快捷键

  • Alt + Insert ,选择自动生成构造方法/set/get方法

插件

  • PTG -> 鼠标右键选择PTG自动生成标准JavaBean类

2’字符串

String字符串

介绍

Stringの背景

  • String类定义在java.lang包下,使用时不需要导包
  • Java中所有字符串字面量都是这个类的对象

Stringの特别注意点

  • 字符串的内容在创建后,就不能修改了!!!

创建String对象的方法

直接赋值

  • String str = "example";

使用new关键字创建String对象

  • public String() -> 没有任何内容,创建空白字符串对象
  • public String(String original) -> 根据传入的内容,创建字符串对象
  • public String(char[] chs) -> 根据字符数组,创建字符串对象
    • 字符串在创建后就不能更改,但是可以利用字符数组修改字符串的内容;先把要更改的字符串转化为字符数组,然后修改字符数组,最后再转化为字符串即可
  • public String(byte[] chs) -> 根据字节数组,创建字符串对象
    • 网络中传输的信息都是字节信息,可以利用这种构造方法将字节信息转化为字符串

键盘录入的字符串也算作new的字符串对象

String内存分析

  1. 直接赋值的字符串内存
    • 存储在堆内存中的串池StringTable中
    • 每次用这种方式创建一个字符串的时候,JVM都会现在串池中查看有没有这个字符串
      • 有的话,就让这个字符串指向串池中的字符串位置,即复用已有字符串
      • 没有的话,就在串池中创建一个该字符串数据,然后让新的字符串指向这个串池中的字符串数据,即创建新的字符串
  1. new创建的字符串内存
    • 在堆里面创建新的空间,用来存放new出来的字符串对象
    • 每次用这种方式创建一个字符串的时候,JVM都会在堆空间中开辟一块新空间,即不可能地址重复

  • int length()方法 -> 返回字符串长度
    • 数组的length是属性不加()
    • 字符串的length是成员方法,要加()

3’集合ArrayList

介绍

为什么要有集合

  • 解决大量数据的难统一管理问题

好处/作用

  • 长度可变,自动扩容
  • 存储内容随意,既可以基本数据类型(使用包装类)也可以引用数据类型
    • 当然这个特点数组也有
  • 存取简单

与数组相比

  • 集合:长度可变,只能存引用数据类型,基本数据类型要用包装类
  • 数组:长度固定,可以存引用数据类型和基本数据类型

分类

  • 巨多,后面学完回来补分类

ArrayList集合(动态数组)

创建方式

集合本质上也是一个类,所以创建集合就是创建类的对象,大体格式不变

格式

ArrayList<E> list = new ArrayList<>();

注意加了一个<E>表示泛型,E就是数据类型(可以是基本,也可以是引用)

比如ArrayList<Integer> list = new ArrayList<>();

打印集合对象名

集合对象名不代表集合对象的地址值

打印的时候不会打印地址,而是以[]的格式,将集合对象中的元素全部打印出来

成员方法

  • 增

boolean add(E e) -> 添加元素,返回值表示是否成功(基本都是true)

  • 删

boolean remove(E e) -> 删除指定元素,返回值表示是否删除成功

E remove(int index) -> 删除指定索引元素,返回被删除元素

  • 改

E set(int index,E e) -> 修改指定索引元素,返回原来的元素

  • 查

E get(int index) -> 获取指定索引元素

int size() -> 集合的长度/元素个数

基本数据类型的包装类

  • byte -> Byte
  • short -> Short
  • int -> Integer
  • long -> Long
  • float -> Float
  • double -> Double
  • char -> Character
  • boolean -> Boolean

5’面向对象进阶

static静态

为什么要用static修饰符?

  • 标注一个类中”共享”的内容

静态变量

被static修饰的成员变量,就叫做静态变量

  • 特点
    • 静态变量被这个类的所有对象共享
    • 静态变量不属于某个对象,而属于这个类
    • 随着类的加载而加载,优先于对象存在
  • 调用方式
    • 类名调用(推荐)Student.teacherName这个属性就是Student类中共享的内容
    • 对象名调用(可以,但是不推荐)stu.teacherName

内存图

p121-15:00开始

静态方法

被static修饰的成员方法,就叫做静态方法

  • 特点
    • 多用在测试类和工具类中,JavaBean类中很少用
  • 调用方式
    • 类名调用(推荐)Student.showTeacher()
    • 对象名调用(不推荐)stu.showTeacher()

工具类

是什么?

  • 能帮帮我们做一些事情,且不是用来描述某个事物的类
    • 比如说Math类就是工具类,里面有很多数学属性和方法供我们调用,来解决问题

书写规则

  • 类名见名知意
  • 私有化构造方法 -> 不让别人创建这个工具类的对象,因为工具类的对象没有实际意义,不需要使用对象
  • 方法定义为静态 -> 方便调用,直接用类名.方法名()即可调用这个方法

static访问权限

  • 静态方法只能访问静态变量/方法,即静态内容
  • 非静态方法可以访问静态/非静态变量&方法,即所有内容
  • 静态方法中没有this关键字
