1’面向对象编程

OOP介绍

是什么

面向对象是什么

面向对象编程, 即OOP(Object-Oriented Programming)

它的核心就是, 程序员所写的代码和代码执行的作用, 都是面向于对象的

什么叫面向于对象? 面向 -> 拿/取, 对象 -> 能干活的东西

对象就可以理解为工具, 然后我们要解决问题的时候, 就要用这个工具来解决对应的问题, 这就可以类比为 : 我们人类解决问题是面向工具的, 那么对于程序而言, 其解决问题就是面向对象的 即 面向对象就是拿能干活的东西来解决问题

面向对象三大特性是什么

封装 -> 封装指的是将数据和操作数据的方法捆绑在一起的机制，以隐藏对象的内部状态并限制对其访问

继承 -> 继承指的是一个类可以使用另一个类的属性和方法，并可以重写或添加新的功能

多态 -> 多态指的是同一个方法可以在不同的对象上产生不同的行为

封装

封装介绍

封装是什么

封装是设计类/对象的规则

它告诉我们 如何正确地设计一个类/对象的属性和方法

目的是将对象的属性和属性对应的行为捆绑在一起, 限制类外内容对其的访问

封装的原则就是 对象代表什么, 就要封装对应的数据, 并提供数据相关的行为

  • 比如,人这个类,和圆这个类,当有一个”画圆”的方法时,应该设计到圆这个类中,因为画圆这个行为属于圆这个类/对象
  • 再比如, 张三砍李四导致李四嗝屁, 嗝屁这个行为就是李四这个对象的行为, 只是张三调用了李四嗝屁的行为, 导致了李四的嗝屁, 本质上还是李四自己的嗝屁行为导致了李四的嗝屁
封装的核心思想是什么

对象代表什么, 就要封装对应的数据, 并且提供数据相关的行为

  • 所以, 封装其实主要针对类中的成员方法和成员属性

而类中的构造方法是为了初始化对象的, 也可以算作封装的一个内容

但是类中的代码块和内部类这两块内容并不属于封装的内容, 而笔者还是将其归类到封装模块下~

目的是使大家能够以封装继承多态的宏观视角理解面向对象, 不要拘泥于细节

封装的学习内容是什么

在这一部分, 笔者将以个人理解, 详细地描述学习封装时, 对应的学习内容

从定义一个类开始, 学习封装就要学习 1.1如何定义类

由于定义类的目的是使用对象(以标准JavaBean为例说明), 而要用对象就要学习 1.2如何创建对象

然后再学习如何使用对象,即 1.3如何使用类和对象

在使用对象的时候, 想要规定谁能调用成员而谁不能调用成员, 就要学习 2权限修饰符

此时就将类/对象中的成员保护进安全屋了, 不是谁都能直接调用它们了, 但是还需要给外部留一个接口, 不然类外部分就彻底无法调用类中成员了, 这个接口就是set和get方法, 所以还要学习 3set/get方法 , 以便对类中的属性进行赋值和访问

而在set方法中, 对应传入的数据, 还要进行正确性校验, 不是什么值都能用于对象属性初始化的, 就衍生出this关键字, 所以就要学习 4this关键字

但是吧, 每次创建对象后我都要手动调用set方法为对象中的属性赋值, 太麻烦了, 所以就衍生出构造方法, 它能在创建类的对象的时候, 就直接为该对象赋值, 简化了数据初始化的流程, 即要学习 5构造方法

完成上面这几部分的学习后, 就完成了封装的学习内容, 此时我们就引入JavaBean类这个概念, 他表示的就是完成封装这一步的描述事物的类, 所以就要学习 6标准JavaBean类

OvO, 到此, 就真的彻底的完成了封装这一面向对象特性的学习, 后续的内容都是基于封装的拓展内容

然后就要学一个static静态修饰符, 它是和工具类绑定的, 用来简化调用方法这一步流程, 从创建对象 -> 使用对象中的方法 简化到 利用类名使用类中的方法, 就要学习 7static静态修饰符 和 8工具类

就下来就是与封装不相关, 但是笔者由归类在封装这一部分的两个内容, 即 9代码块 和 10内部类

封装是设计类/对象中属性和方法的规则, 其告诉我们如何设计好一个标准JavaBean类, 没有这个过程就没有设计类/对象的标准, 没有标准成何体统~

类和对象

类和对象是什么

简单的描述就是

  • 类是一张设计图纸, 利用这个图纸我们可以生产出产品
  • 对象就是生产出的产品, 而我们可以使用这个产品来解决问题

而用行话来描述就是

  • 类是对象共同特征的描述, 即抽象化
  • 对象是类产生的实例, 即实例化
类的五个组成部分是什么

组成部分如下

  • 成员属性
  • 成员方法
  • 构造方法
  • 代码块
  • 内部类

它们与封装的关系是

  • 成员属性和成员方法是封装的核心组成部分。

  • 构造方法用于初始化对象, 可以理解为封装的组成部分

  • 而代码块和内部类则提供了一些额外的机制来组织代码和实现封装, 但它们并不是封装的必要组成部分

this关键字

就近原则是什么

谁离我近, 我就用谁

比如有两个同名变量number, 一个在全局作用, 一个在函数体内部作用, 那当代码执行到函数体内部的时候, 如果没有特殊声明, 那肯定用的是函数体中的number变量, 因为它离得近

由于就近原则的特点, 在类中书写方法涉及到对属性的操作时, 就要注意是否混淆了全局变量(即成员属性)和局部变量(即方法中定义的变量)

构造方法

构造方法是什么

构造方法是什么

  • 也称构造器,构造函数

作用是

  • 在创建对象的时候, 由JVM自动调用, 自动给成员变量(属性)进行初始化操作

    就不用手动调用set方法进行赋值了

为什么要有构造方法

使用构造方法, 就不用手动调用set方法进行赋值了

如何理解构造方法

构造方法 ≠ 用来创建对象

Student student = new Student()

真正创建对象这一步, 是 new 关键字干的, 不是构造方法干的

构造方法只是创建对象之中的一步, 在这一步进行成员变量的初始化

如何定义构造方法

定义格式

定义要求

  • 构造方法名与类名完全相同
  • 构造方法没有返回值类型,连void空类型都没有
  • 构造方法没有具体返回值(不可以写return语句)

空参构造细节

  • 如果类中没有写构造方法,JVM会自动添加一个空参构造

    如果定义了构造方法,JVM就不会自动添加默认的空参构造了

    所以空参构造和有参构造都必须写上(不管用不用都要写)

  • 空参构造和有参构造就是构造方法的重载

如何使用构造方法

如何使用

构造方法不用手动调用, 在创建对象时由JVM自动调用执行, 每创建一个对象就会自动调用一次构造方法

String字符串

介绍

Stringの背景

  • String类定义在java.lang包下,使用时不需要导包
  • Java中所有字符串字面量都是这个类的对象

Stringの特别注意点

  • 字符串的内容在创建后,就不能修改了!!!

创建String对象的方法

直接赋值

  • String str = "example";

使用new关键字创建String对象

  • public String() -> 没有任何内容,创建空白字符串对象
  • public String(String original) -> 根据传入的内容,创建字符串对象
  • public String(char[] chs) -> 根据字符数组,创建字符串对象
    • 字符串在创建后就不能更改,但是可以利用字符数组修改字符串的内容;先把要更改的字符串转化为字符数组,然后修改字符数组,最后再转化为字符串即可
  • public String(byte[] chs) -> 根据字节数组,创建字符串对象
    • 网络中传输的信息都是字节信息,可以利用这种构造方法将字节信息转化为字符串

键盘录入的字符串也算作new的字符串对象

String内存分析

  1. 直接赋值的字符串内存
    • 存储在堆内存中的串池StringTable中
    • 每次用这种方式创建一个字符串的时候,JVM都会现在串池中查看有没有这个字符串
      • 有的话,就让这个字符串指向串池中的字符串位置,即复用已有字符串
      • 没有的话,就在串池中创建一个该字符串数据,然后让新的字符串指向这个串池中的字符串数据,即创建新的字符串
  1. new创建的字符串内存
    • 在堆里面创建新的空间,用来存放new出来的字符串对象
    • 每次用这种方式创建一个字符串的时候,JVM都会在堆空间中开辟一块新空间,即不可能地址重复

介绍

为什么要有集合

  • 解决大量数据的难统一管理问题

好处/作用

  • 长度可变,自动扩容
  • 存储内容随意,既可以基本数据类型(使用包装类)也可以引用数据类型

与数组相比

  • 数组:长度固定,可以存引用数据类型和基本数据类型

分类

  • 巨多,后面学完回来补分类

注意加了一个<E>表示泛型,E就是数据类型(可以是基本,也可以是引用)

比如ArrayList<Integer> list = new ArrayList<>();

打印集合对象名

集合对象名不代表集合对象的地址值

打印的时候不会打印地址,而是以[]的格式,将集合对象中的元素全部打印出来

5’面向对象进阶

static静态

为什么要用static修饰符?

  • 标注一个类中”共享”的内容

静态变量

被static修饰的成员变量,就叫做静态变量

  • 特点
    • 静态变量被这个类的所有对象共享
    • 静态变量不属于某个对象,而属于这个类
    • 随着类的加载而加载,优先于对象存在
  • 调用方式
    • 类名调用(推荐)Student.teacherName这个属性就是Student类中共享的内容
    • 对象名调用(可以,但是不推荐)stu.teacherName

静态方法

被static修饰的成员方法,就叫做静态方法

  • 特点
    • 多用在测试类和工具类中,JavaBean类中很少用
  • 调用方式
    • 类名调用(推荐)Student.showTeacher()
    • 对象名调用(不推荐)stu.showTeacher()

工具类

是什么?

  • 能帮帮我们做一些事情,且不是用来描述某个事物的类
    • 比如说Math类就是工具类,里面有很多数学属性和方法供我们调用,来解决问题

书写规则

  • 类名见名知意
  • 私有化构造方法 -> 不让别人创建这个工具类的对象,因为工具类的对象没有实际意义,不需要使用对象
  • 方法定义为静态 -> 方便调用,直接用类名.方法名()即可调用这个方法

static访问权限

  • 静态方法只能访问静态变量/方法,即静态内容
  • 非静态方法可以访问静态/非静态变量&方法,即所有内容
  • 静态方法中没有this关键字
