1’面向对象编程

OOP介绍

是什么

面向对象是什么

面向对象编程, 即OOP(Object-Oriented Programming)

它的核心就是, 程序员所写的代码和代码执行的作用, 都是面向于对象的

什么叫面向于对象? 面向 -> 拿/取, 对象 -> 能干活的东西

对象就可以理解为工具, 然后我们要解决问题的时候, 就要用这个工具来解决对应的问题, 这就可以类比为 : 我们人类解决问题是面向工具的, 那么对于程序而言, 其解决问题就是面向对象的 即 面向对象就是拿能干活的东西来解决问题

面向对象三大特性是什么

封装 -> 封装指的是将数据和操作数据的方法捆绑在一起的机制，以隐藏对象的内部状态并限制对其访问

继承 -> 继承指的是一个类可以使用另一个类的属性和方法，并可以重写或添加新的功能

多态 -> 多态指的是同一个方法可以在不同的对象上产生不同的行为

封装

封装介绍

封装是什么

封装是设计类/对象的规则

它告诉我们 如何正确地设计一个类/对象的属性和方法

目的是将对象的属性和属性对应的行为捆绑在一起, 限制类外内容对其的访问

封装的原则就是 对象代表什么, 就要封装对应的数据, 并提供数据相关的行为

  • 比如,人这个类,和圆这个类,当有一个”画圆”的方法时,应该设计到圆这个类中,因为画圆这个行为属于圆这个类/对象
  • 再比如, 张三砍李四导致李四嗝屁, 嗝屁这个行为就是李四这个对象的行为, 只是张三调用了李四嗝屁的行为, 导致了李四的嗝屁, 本质上还是李四自己的嗝屁行为导致了李四的嗝屁
封装的核心思想是什么

对象代表什么, 就要封装对应的数据, 并且提供数据相关的行为

  • 所以, 封装其实主要针对类中的成员方法和成员属性

而类中的构造方法是为了初始化对象的, 也可以算作封装的一个内容

但是类中的代码块和内部类这两块内容并不属于封装的内容, 而笔者还是将其归类到封装模块下~

目的是使大家能够以封装继承多态的宏观视角理解面向对象, 不要拘泥于细节

封装的学习内容是什么

在这一部分, 笔者将以个人理解, 详细地描述学习封装时, 对应的学习内容

从定义一个类开始, 学习封装就要学习 1.1如何定义类

由于定义类的目的是使用对象(以标准JavaBean为例说明), 而要用对象就要学习 1.2如何创建对象

然后再学习如何使用对象,即 1.3如何使用类和对象

在使用对象的时候, 想要规定谁能调用成员而谁不能调用成员, 就要学习 2权限修饰符

此时就将类/对象中的成员保护进安全屋了, 不是谁都能直接调用它们了, 但是还需要给外部留一个接口, 不然类外部分就彻底无法调用类中成员了, 这个接口就是set和get方法, 所以还要学习 3set/get方法 , 以便对类中的属性进行赋值和访问

而在set方法中, 对应传入的数据, 还要进行正确性校验, 不是什么值都能用于对象属性初始化的, 就衍生出this关键字, 所以就要学习 4this关键字

但是吧, 每次创建对象后我都要手动调用set方法为对象中的属性赋值, 太麻烦了, 所以就衍生出构造方法, 它能在创建类的对象的时候, 就直接为该对象赋值, 简化了数据初始化的流程, 即要学习 5构造方法

完成上面这几部分的学习后, 就完成了封装的学习内容, 此时我们就引入JavaBean类这个概念, 他表示的就是完成封装这一步的描述事物的类, 所以就要学习 6标准JavaBean类

OvO, 到此, 就真的彻底的完成了封装这一面向对象特性的学习, 后续的内容都是基于封装的拓展内容

然后就要学一个static静态修饰符, 它是和工具类绑定的, 用来简化调用方法这一步流程, 从创建对象 -> 使用对象中的方法 简化到 利用类名使用类中的方法, 就要学习 7static静态修饰符 和 8工具类

就下来就是与封装不相关, 但是笔者由归类在封装这一部分的两个内容, 即 9代码块 和 10内部类

类和对象

类和对象是什么

简单的描述就是

  • 类是一张设计图纸, 利用这个图纸我们可以生产出产品
  • 对象就是生产出的产品, 而我们可以使用这个产品来解决问题

而用行话来描述就是

  • 类是对象共同特征的描述, 即抽象化
  • 对象是类产生的实例, 即实例化
类的五个组成部分是什么

组成部分如下

  • 成员属性
  • 成员方法
  • 构造方法
  • 代码块
  • 内部类

它们与封装的关系是

  • 成员属性和成员方法是封装的核心组成部分。

  • 构造方法用于初始化对象, 可以理解为封装的组成部分

  • 而代码块和内部类则提供了一些额外的机制来组织代码和实现封装, 但它们并不是封装的必要组成部分

创建对象的内存分析

本篇主要理解, 创建一个对象时的内存分析

Java内存分配

Java虚拟机JVM会和其他应用程序(比如说QQ, 网页)一样, 占用内存中的部分空间, 以执行程序

而为了更好地管理这块内存空间, JVM将其占用的内存分为5个部分↓

  • 栈

    方法被调用时, 需要进栈执行, 其中的变量也会进行初始化, 等方法体执行完毕后, 方法就要出栈, 其中声明的临时存储在栈空间中的变量也会销毁

  • 堆

    所有被new出来的内容, 都存储在堆空间中, 这些被new出来的内容会有一个地址值表示其在堆空间中的位置

  • 方法区(元空间)

    前世今生

    • jdk7以前, 方法区和堆是连在一起的, 而这种设计并不好

      jdk8之后, 新增了元空间, 取消方法区, 将元空间和堆分开, 将原来方法区中的部分功能分到元空间, 还有一部分分到堆空间

      但是在篇内存分析的讲解中, 笔者还是将元空间命名为原本的方法区, 便于理解

    功能

    • 运行一个类的时候, 这个类的字节码文件就会加载到方法区中, 临时存储

  • 本地方法栈

  • 寄存器

一个对象的内存分析

比如说创建一个Student类的对象student Student student = new Student();

在JVM分配的内存中, 至少要执行以下5步

  1. 在方法区中加载Student.class字节码文件, 临时存储类中的方法

  2. 方法入栈, 在栈空间中声明局部变量student

    这个student变量存储的内容其实就是将要赋给它的在堆空间中的地址值

  3. 在堆空间中开辟一块空间, 用来存储student对象中的内容

    它在堆空间中的位置就是它的地址值

  4. 初始化

    默认/显式/构造方法初始化都是为student对象中的成员变量进行赋值/初始化

    默认初始化, 就是基本数据类型赋初始值(0, 0.0, ‘\u0000’, false), 引用数据类型赋null值

    显式初始化, 就是在类中定义成员变量的时候直接赋值了, 比如private int age =20; , 那么默认初始值就会被其覆盖

    构造方法初始化, 就是创建类的对象时, 调用有参构造将参数赋给对象的成员变量

  5. 将堆空间中的地址值赋值给栈空间中的局部变量student

以实际例子来说明, 比如现在有一个Student类(①), 然后要在Test类(②)的main方法中, 执行 Student s = new Student(); 这条创建对象语句, 那就要走5步流程

  1. 方法区加载Test类的字节码文件Test.class, 临时存储其中的main方法

  2. main方法入栈, (↑↑想要声明Student的变量s, 就要先在方法区中再加载Student类的字节码文件Student.class, 临时存储类的成员属性和方法), 然后在栈空间中声明并存储临时变量s

  3. 然后在堆空间中开辟空间分配给s对象, 这块内存里面就存储s对象中的成员属性的值, 和成员方法的地址(指向方法区中存储的方法)

  4. 对s对象进行初始化

    此案例中采用默认初始化

  5. 将堆空间中s对象地址值赋给栈空间中s变量

两个对象的内存分析

在上述案例创建s对象的基础上, 再创建一个s2对象

那么还是走5步流程

  1. 加载Test.class, Studnet.class字节码文件, 临时存储方法
  2. main方法入栈, 声明临时变量s和s2
  3. 堆空间中开辟两块空间, 分给s对象和s2对象
  4. 对这两个对象初始化
  5. 将s和s2对象在堆空间中的地址值赋给栈空间中的s和s2变量

两个对象变量指向同一块堆空间地址

即Student s1 = new Student(); 然后 Student s2 = s1;

在第二创建对象语句时, 就会将s1在堆空间中的地址值赋给s2变量, 这样两个对象变量指向同一块空间, 任何一方执行的操作都会影响堆空间中存储的数据, 间接影响到另一个对象

this关键字

就近原则是什么

谁离我近, 我就用谁

比如有两个同名变量number, 一个在全局作用, 一个在函数体内部作用, 那当代码执行到函数体内部的时候, 如果没有特殊声明, 那肯定用的是函数体中的number变量, 因为它离得近

由于就近原则的特点, 在类中书写方法涉及到对属性的操作时, 就要注意是否混淆了全局变量(即成员属性)和局部变量(即方法中定义的变量)

this关键字是什么

this关键字本质就是当前对象的地址值(即方法的调用者的地址值)

作用是用this指向的变量表示类中的成员属性变量, 而不是传入的局部变量, 可以 区分成员变量和局部变量

this关键字内存分析

一个类中的this关键字的本质是 方法调用者的地址值

比如说main方法中创建了s对象 Student s = new Student(); 然后调用s对象中的show方法 s.show(); 而show方法中的内容就是输出s对象的姓名,学号,班级(都用this关键字指向了)

此时调用show方法遇到this关键字, 就会被解析为是s对象的地址值, 就要去s对象的堆空间中取出姓名,学号,班级信息

笔者个人理解, 其实可以把this关键字(存储某个对象地址值)看成一个类中的成员变量, 哪个对象调用类中的方法, 这个this存储的就是这个对象的地址值

成员&局部变量的区别

成员变量 -> 类中的属性(在方法体之外)

​ 比如Student类中定义的int age; String name;都是成员变量

局部变量 -> 方法中的变量(在方法体之内)

​ 比如setAge方法中定义的形参(setAge(int age)中的age), 这就是一个局部变量, 方法被调用后, 执行完毕(出栈)后就会被销毁

二者区别如下(不用记)

区别 成员变量 局部变量
类中位置不同 类中, 方法体之外 类中, 方法体之内
初始化值不同 有默认初始化值 没有默认初值, 需要自行赋值
内存位置不同 堆内存空间 栈内存空间
生命周期不同 随着对象的创建而创建, 随着对象的销毁而销毁 随着方法的调用而产生, 随着方法的出栈(运行完毕)而销毁
作用域不同 整个类中有效 当前方法体中有效

构造方法

构造方法是什么

构造方法是什么

  • 也称构造器,构造函数

作用是

  • 在创建对象的时候, 由JVM自动调用, 自动给成员变量(属性)进行初始化操作

    就不用手动调用set方法进行赋值了

为什么要有构造方法

使用构造方法, 就不用手动调用set方法进行赋值了

如何理解构造方法

构造方法 ≠ 用来创建对象

Student student = new Student()

真正创建对象这一步, 是 new 关键字干的, 不是构造方法干的

构造方法只是创建对象之中的一步, 在这一步进行成员变量的初始化

如何定义构造方法

定义格式

定义要求

  • 构造方法名与类名完全相同
  • 构造方法没有返回值类型,连void空类型都没有
  • 构造方法没有具体返回值(不可以写return语句)

空参构造细节

  • 如果类中没有写构造方法,JVM会自动添加一个空参构造

    如果定义了构造方法,JVM就不会自动添加默认的空参构造了

    所以空参构造和有参构造都必须写上(不管用不用都要写)

用 双引号””包裹的内容 都是字符串

  • 其中可以存储的内容是任意个字符

而且字符串还可以通过 + 运算符, 实现字符串的拼接操作, 还有n个字符串的操作

  • Java程序中所有字符串文字都是String类的对象

String创建对象的内存分析

Java虚拟机的内存模型

栈 -> 和方法运行有关, 进栈&出栈

堆 -> 和new出来的对象有关, new出来都存储在堆空间

  • 串池(StringTable) -> 可以理解为字符串常量池

    用来存字符串对象, 但是 只存储直接赋值方式创建的字符串

    JDK7以前串池放在方法区当中, JDK7之后就放到堆空间了

方法区(元空间) -> 加载类的字节码文件, 临时存储方法

直接赋值的字符串内存

  • 存储在堆内存中的 串池StringTable 中

  • 每次用这种方式创建一个字符串的时候, JVM都会现在串池中查看有没有这个字符串

    有的话, 就让这个字符串指向串池中的字符串位置, 即复用已有字符串

    没有的话, 就在串池中创建一个该字符串数据, 然后让新的字符串指向这个串池中的字符串数据, 即创建新的字符串

所以创建字符串对象, 还是多多用直接赋值的方式, 代码又简单, 效率又高(节约内存)

new创建的字符串内存

  • 在堆里面开辟新的空间, 用来存放new出来的字符串对象
  • 每次用这种方式创建一个字符串的时候, JVM都会在堆空间中开辟一块新空间, 即不可能地址重复

字符串容器StringBuilder是什么

StringBuilder是一个字符串容器, 其内容是可变的

sb对象中存储的内容默认都是字符串类型

  • 向它添加数据时，如果数据不是字符串类型，它们会自动将其转换为字符串类型并存储

作用

  • 用来对字符串String对象进行操作, 提高字符串操作效率

    对 字符串拼接 和 字符串反转 的操作极为方便, 且效率也高

链式编程是什么

在使用sb对象的方法时, 可以写成这种形式 sb.append(1).append(2).reverse().toString(); ,这种思想就是链式编程

链式 -> 一串一串连接

链式编程 -> 在调用方法的时候, 暂时不用变量接收结果, 而是继续再调用方法, ..以此类推

为什么要学字符串String

字符串是一种常用的数据类型, 需要其处理多个字符组成的集合, 没有字符串String, 就没有便捷快速的处理多字符的方法

为什么要学字符串容器StringBuilder

字符串String对于存储内容的要求较高, 一旦创建其内容就不可改变, 而使用StringBuilder可以 解决不可更改内容 的这个问题

并且, StringBuilder对于字符串的 操作效率也很高(主要是拼接和反转)

比如有s1,2,3,4,5,6这六个字符串, 我想要将它们拼接在一起返回到str对象, 只用String类的话, 就要一个一个拼接字符串, 比如s1+s2的时候, 就会创建 s1+s2 这个之后不会用到的字符串对象, 效率低下占用内存

而用StringBuilder类的话, 就直接把这几个字符串添加到sb对象容器中, 最后转为String对象就完成了, 效率高

为什么要学字符串工具StringJoiner

使字符串拼接操作更方便

可以指定分隔符, 起始符号和结束符号, 以快速构建包含若干字符串的字符串序列

  • 集合:长度可变,只能存引用数据类型,基本数据类型要用包装类
  • 数组:长度固定,可以存引用数据类型和基本数据类型

格式

注意加了一个<E>表示泛型,E就是数据类型(可以是基本,也可以是引用)

比如ArrayList<Integer> list = new ArrayList<>();

打印集合对象名

集合对象名不代表集合对象的地址值

  • 查

static静态

为什么要用static修饰符?

  • 标注一个类中”共享”的内容

静态变量

被static修饰的成员变量,就叫做静态变量

  • 特点
    • 静态变量被这个类的所有对象共享
    • 静态变量不属于某个对象,而属于这个类
    • 随着类的加载而加载,优先于对象存在
  • 调用方式
    • 类名调用(推荐)Student.teacherName这个属性就是Student类中共享的内容
    • 对象名调用(可以,但是不推荐)stu.teacherName

静态方法

被static修饰的成员方法,就叫做静态方法

  • 特点
    • 多用在测试类和工具类中,JavaBean类中很少用
  • 调用方式
    • 类名调用(推荐)Student.showTeacher()
    • 对象名调用(不推荐)stu.showTeacher()

工具类

是什么?

  • 能帮帮我们做一些事情,且不是用来描述某个事物的类
    • 比如说Math类就是工具类,里面有很多数学属性和方法供我们调用,来解决问题

书写规则

  • 类名见名知意
  • 私有化构造方法 -> 不让别人创建这个工具类的对象,因为工具类的对象没有实际意义,不需要使用对象
  • 方法定义为静态 -> 方便调用,直接用类名.方法名()即可调用这个方法

static访问权限

  • 静态方法只能访问静态变量/方法,即静态内容
  • 非静态方法可以访问静态/非静态变量&方法,即所有内容
  • 静态方法中没有this关键字
